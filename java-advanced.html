Java Advanced

Java'ning kuchli vositalari: Exception Handling, Generics, Lambda, Stream API, Optional, Enum va Annotations.

Dars maqsadi: Professional Java dasturchisi uchun zarur bo'lgan ilg'or tushunchalarni amaliy misollar va nazariy chuqurlik bilan o'rganamiz.

Exception Handling — kirish

Dasturda kutilmagan vaziyatlar doimo bo'ladi: fayl topilmadi, internet uzildi, raqamga matn keldi va h.k. Exception Handling bu vaziyatlarni boshqarish vositasi. 

// Exception bo'lmagan kod
int[] arr = {1, 2, 3};
System.out.println(arr[10]);  // CRASH! ArrayIndexOutOfBoundsException

Exception — bu Java'da Object. Xato sodir bo'lganda yaratiladi va "tashlanadi" (throw).

Agar uni "tutib olmasak" (catch), dastur to'xtab qoladi. Stack trace esa qaerda xato bo'lganini ko'rsatadi.

Exception ierarxiyasi

Java'da barcha xatolar ildizi — Throwable. Undan ikki shox chiqadi:

Throwable
├── Error              ← OutOfMemory, StackOverflow (tutilmaydi)
└── Exception
    ├── IOException             ← CHECKED (compile-time)
    ├── SQLException            ← CHECKED
    └── RuntimeException        ← UNCHECKED
        ├── NullPointerException
        ├── ArrayIndexOutOfBoundsException
        ├── ArithmeticException
        └── IllegalArgumentException

Checked

Compile vaqtida tekshiriladi. Albatta try-catch yoki throws bilan boshqarish kerak.

Unchecked

RuntimeException avlodlari. Compile tekshirmaydi — runtime'da paydo bo'ladi.

Error

Tizim darajasidagi muammolar. Odatda tutib olinmaydi (JVM hal qiladi).

try-catch-finally

Exception'ni tutib olish va boshqarish uchun asosiy konstruksiya.

try {
    int[] arr = {1, 2, 3};
    System.out.println(arr[10]);
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Indeks chegaradan tashqarida: " + e.getMessage());
} catch (Exception e) {
    System.out.println("Boshqa xato: " + e.getMessage());
} finally {
    System.out.println("Bu BLOK HAR DOIM bajariladi");
}

// Java 7+ Multi-catch
try {
    // kod
} catch (IOException | SQLException e) {
    System.out.println("IO yoki SQL xato: " + e);
}
finally blok har doim ishlaydi — exception bo'ladimi yoki yo'qmi. Asosan resurslarni yopish uchun (fayl, DB ulanish).

throw vs throws va try-with-resources

// THROW — exception yaratish va tashlash
public void setAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException("Yosh manfiy bo'lmaydi: " + age);
    }
    this.age = age;
}

// THROWS — metod exception tashlashi mumkinligini e'lon qilish
public void readFile(String path) throws IOException {
    Files.readAllLines(Paths.get(path));
}

try-with-resources — Java 7+, resurs avtomatik yopiladi:

// Eski usul (uzun)
BufferedReader reader = null;
try {
    reader = new BufferedReader(new FileReader("file.txt"));
    String line = reader.readLine();
} finally {
    if (reader != null) reader.close();
}

// Yangi usul (qisqa, xavfsiz)
try (BufferedReader reader = new BufferedReader(new FileReader("file.txt"))) {
    String line = reader.readLine();
}
// reader avtomatik yopiladi — finally yozish shart emas!

Custom Exception

O'z exception'imizni yaratishimiz mumkin. Bizning domenga oid xatolar uchun.

// Checked exception
public class InsufficientBalanceException extends Exception {
    public InsufficientBalanceException(String message) {
        super(message);
    }
}

// Unchecked exception
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(long userId) {
        super("Foydalanuvchi topilmadi: " + userId);
    }
}

// Foydalanish
public void withdraw(double amount) throws InsufficientBalanceException {
    if (amount > balance) {
        throw new InsufficientBalanceException("Balans yetarli emas");
    }
    balance -= amount;
}
Qoida: Custom exception nomi Exception bilan tugashi tavsiya etiladi. Checked — biznes mantiqi xatolari. Unchecked — dasturchi xatosi.

Xavfsiz bo'lish (Safe Division)

Metod ikkita son qabul qilsin va bo'linmasini qaytarsin. b == 0 bo'lsa Optional.empty() qaytarsin, exception'ni "yutib" yuborsin.

Misol
safeDivide(10, 2) → Optional[5]
safeDivide(10, 0) → Optional.empty 
public static Optional<Integer> safeDivide(int a, int b) {
    try {
        return Optional.of(a / b);
    } catch (ArithmeticException e) {
        return Optional.empty();
    }
}

System.out.println(safeDivide(10, 2));  // Optional[5]
System.out.println(safeDivide(10, 0));  // Optional.empty

⚡ Masala 3 Medium

String'dan int parse (Robust)

Metod string qabul qilib, butun songa aylantirsin. Agar parse bo'lmasa — defaultValue qaytarsin. null va bo'sh string'lar ham hisobga olinsin.

Misol
parse("123", 0) → 123
parse("abc", 0) → 0
parse(null, -1) → -1
parse("", -1) → -1 
public static int safeParse(String s, int defaultValue) {
    if (s == null || s.isEmpty()) {
        return defaultValue;
    }
    try {
        return Integer.parseInt(s.trim());
    } catch (NumberFormatException e) {
        return defaultValue;
    }
}

System.out.println(safeParse("123", 0));  // 123
System.out.println(safeParse("abc", 0));  // 0
System.out.println(safeParse(null, -1));  // -1
⚡ Masala 4 Hard

Retry mexanizmi

Metod yozing: berilgan operatsiyani N marta urinib ko'rsin. Agar muvaffaqiyatli bo'lsa — natijani qaytarsin. Hammasi muvaffaqiyatsiz bo'lsa — oxirgi exception'ni tashlasin.

Misol
retry(() -> doSomething(), 3) → 3 marta urinadi
Birinchi muvaffaqiyatda to'xtaydi
Hammasi muvaffaqiyatsiz → exception 
public static <T> T retry(Supplier<T> operation, int times) throws Exception {
    Exception lastException = null;
    for (int i = 0; i < times; i++) {
        try {
            return operation.get();
        } catch (Exception e) {
            lastException = e;
            System.out.println("Urinish " + (i+1) + " muvaffaqiyatsiz: " + e.getMessage());
        }
    }
    throw lastException;
}

// Foydalanish
String result = retry(() -> fetchFromAPI(), 3);

Generics — nima va nima uchun?

Generics — turlarni parametrlash. Bitta klass yoki metodni turli tiplar bilan ishlatish imkonini beradi. 

// Generics SIZ
List list = new ArrayList();
list.add("Salom");
String s = (String) list.get(0); // CASTING kerak, xato bo'lishi mumkin

// Generics BILAN
List<String> list2 = new ArrayList<>();
list2.add("Salom");
String s2 = list2.get(0); // Tur xavfsiz, casting yo'q!

Type Erasure: Java'da Generics compile vaqtida tekshiriladi va keyin o'chiriladi. JVM darajasida List<String> va List<Integer> bir xil — oddiy List!

Bu eski kod bilan moslik (backward compatibility) uchun qilingan.

Generic class va method

// Generic class
public class Box<T> {
    private T value;

    public Box(T value) { this.value = value; }
    public T get() { return value; }
    public void set(T value) { this.value = value; }
}

// Foydalanish
Box<String> stringBox = new Box<>("Salom");
Box<Integer> intBox = new Box<>(42);

// Generic method
public static <T> void printArray(T[] arr) {
    for (T item : arr) {
        System.out.println(item);
    }
}

printArray(new String[]{"A", "B", "C"});
printArray(new Integer[]{1, 2, 3});

// Bir nechta type parameter
public class Pair<K, V> {
    K key;
    V value;
}
Konvensiya: T — Type, E — Element, K — Key, V — Value, N — Number.

Bounded Types va Wildcards

// Bounded type — faqat Number va uning avlodlari
public <T extends Number> double sum(List<T> list) {
    double total = 0;
    for (T num : list) {
        total += num.doubleValue();  // Number metodlari mavjud
    }
    return total;
}

sum(List.of(1, 2, 3));      // OK (Integer)
sum(List.of(1.5, 2.5));     // OK (Double)
// sum(List.of("a", "b"));  // XATO — String Number'dan emas

Wildcards (?):

List<?> — har qanday turdagi list (read-only)

List<? extends Number> — Number yoki uning child'lari (PECS: Producer Extends)

List<? super Integer> — Integer yoki uning parent'lari (PECS: Consumer Super)

⚡ Masala Medium

Generic Pair klassi

Pair<K, V> klassini yarating. Ikkita element saqlasin. Metodlar:

  • getKey(), getValue()
  • swap() — Pair<V, K> qaytarsin
  • toString() — (key, value) formatda
Misol
Pair<String, Integer> p = new Pair<>("yosh", 25)
p.swap() → Pair<Integer, String>(25, "yosh") 
public class Pair<K, V> {
    private final K key;
    private final V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }

    public Pair<V, K> swap() {
        return new Pair<>(value, key);
    }

    @Override
    public String toString() {
        return "(" + key + ", " + value + ")";
    }
}

Predicate bilan filtr

List of integers'dan juft sonlarni ajratib oluvchi metod yozing — Predicate<Integer> bilan.

Misol
filter([1,2,3,4,5,6], n -> n % 2 == 0) → [2, 4, 6]
filter([1,2,3,4,5], n -> n > 3) → [4, 5] 
public static <T> List<T> filter(List<T> list, Predicate<T> pred) {
    List<T> result = new ArrayList<>();
    for (T item : list) {
        if (pred.test(item)) {
            result.add(item);
        }
    }
    return result;
}

// Foydalanish
List<Integer> nums = List.of(1, 2, 3, 4, 5, 6);
System.out.println(filter(nums, n -> n % 2 == 0)); // [2, 4, 6]

Function bilan transform

List of strings'ni Function bilan o'zgartiring — har bir string'ni katta harfga aylantiring.

Misol
transform(["ali", "vali"], String::toUpperCase)
→ ["ALI", "VALI"] 
public static <T, R> List<R> transform(List<T> list, Function<T, R> fn) {
    List<R> result = new ArrayList<>();
    for (T item : list) {
        result.add(fn.apply(item));
    }
    return result;
}

// Foydalanish
List<String> names = List.of("ali", "vali");
System.out.println(transform(names, String::toUpperCase));
// [ALI, VALI]

// Method reference: String::toUpperCase === s -> s.toUpperCase()

Compose funksiyalari

Ikki Function'ni birlashtiring: g(f(x)). Ya'ni avval birinchi, keyin uning natijasiga ikkinchi.

Misol
f = x -> x + 1
g = x -> x * 2
compose(f, g).apply(3) → g(f(3)) = g(4) = 8 
public static <A, B, C> Function<A, C> compose(
        Function<A, B> f,
        Function<B, C> g) {
    return a -> g.apply(f.apply(a));
}

// Yoki Function ichidagi tayyor andThen ishlatish
Function<Integer, Integer> addOne = x -> x + 1;
Function<Integer, Integer> doubleIt = x -> x * 2;

Function<Integer, Integer> combined = addOne.andThen(doubleIt);
System.out.println(combined.apply(3)); // 8
⚡ Masala 4 Hard

Memoization (Cache funksiyasi)

Funksiyani cachelab ishlatadigan wrapper yarating: bir xil input uchun bir marta hisoblansin, keyingi safarlar saqlangan natijani qaytarsin.

Misol
memoize(n -> { System.out.println("hisoblash..."); return n*n; })
.apply(5) → "hisoblash..." 25
.apply(5) → 25 (qayta hisoblamaydi) 
public static <T, R> Function<T, R> memoize(Function<T, R> fn) {
    Map<T, R> cache = new HashMap<>();
    return input -> cache.computeIfAbsent(input, fn);
}

// Foydalanish
Function<Integer, Integer> expensiveSquare = n -> {
    System.out.println("hisoblash: " + n);
    return n * n;
};

Function<Integer, Integer> cached = memoize(expensiveSquare);
cached.apply(5);  // hisoblash: 5, → 25
cached.apply(5);  // → 25 (cache'dan)
cached.apply(3);  // hisoblash: 3, → 9

Stream API — nima va qanday ishlaydi?

Stream — ma'lumotlar oqimi. Collection ustida funksional usulda ishlash imkonini beradi.

Lazy Evaluation: Stream operatsiyalari "kechiktirilgan" — terminal operatsiya (collect, forEach) chaqirilmaguncha hech narsa ishlamaydi.

Pipeline: Source → Intermediate ops (lazy) → Terminal op (eager) → Result.

Stream != Collection: Stream ma'lumot saqlamaydi, faqat ulardan oqib o'tadi. Bir marta ishlatiladi.

List<Integer> nums = List.of(1, 2, 3, 4, 5, 6);

int sum = nums.stream()                  // Source
    .filter(n -> n % 2 == 0)             // Intermediate (lazy)
    .mapToInt(Integer::intValue)         // Intermediate (lazy)
    .sum();                              // Terminal (eager)
// 2 + 4 + 6 = 12

System.out.println(sum); // 12

Intermediate operations

Stream'ni o'zgartiradi, lekin yangi Stream qaytaradi. Stream → Stream. 

List<String> names = List.of("Ali", "Vali", "Akobir", "Sardor");

// filter — shartga mos elementlar
names.stream()
    .filter(n -> n.length() > 3)  // 4+ harfli
    .forEach(System.out::println);
// Vali, Akobir, Sardor

// map — har bir elementni o'zgartirish
names.stream()
    .map(String::toUpperCase)
    .forEach(System.out::println);
// ALI, VALI, AKOBIR, SARDOR

// sorted — saralash
names.stream().sorted().forEach(System.out::println);

// distinct — takrorlanmas
List.of(1, 2, 2, 3, 3, 4).stream().distinct().forEach(System.out::println);
// 1, 2, 3, 4

// limit, skip — chegaralash
nums.stream().limit(3);  // birinchi 3 ta
nums.stream().skip(2);   // birinchi 2 tasini tashlash

Terminal operations

Stream'ni tugatadi va natijani qaytaradi. Pipeline shu yerda ishlaydi. 

List<Integer> nums = List.of(1, 2, 3, 4, 5);

// collect — Collection'ga to'plash
List<Integer> evens = nums.stream()
    .filter(n -> n % 2 == 0)
    .collect(Collectors.toList());

// count — sanash
long count = nums.stream().filter(n -> n > 2).count();  // 3

// reduce — yig'ish
int sum = nums.stream().reduce(0, Integer::sum);        // 15
Optional<Integer> max = nums.stream().max(Integer::compare);

// findFirst, findAny
Optional<Integer> first = nums.stream().filter(n -> n > 3).findFirst();

// anyMatch, allMatch, noneMatch
boolean hasEven = nums.stream().anyMatch(n -> n % 2 == 0);  // true
boolean allPositive = nums.stream().allMatch(n -> n > 0);   // true

// forEach — har biri uchun amal
nums.stream().forEach(System.out::println);

// Collectors.groupingBy, joining, partitioningBy
Map<Boolean, List<Integer>> parted = nums.stream()
    .collect(Collectors.partitioningBy(n -> n % 2 == 0));
// {false=[1,3,5], true=[2,4]}

Stream bilan group va count

public static Map<Integer, Long> groupByLength(List<String> list) {
    return list.stream()
        .collect(Collectors.groupingBy(
            String::length,
            Collectors.counting()
        ));
}

Talabalarni Stream bilan filtr

List of Students. GPA ≥ 4.0 bo'lganlarni tanlang, ismi bo'yicha tartibga soling va ismlarini vergul bilan ajratilgan string sifatida qaytaring.

Misol

public static String topStudents(List<Student> students) {
    return students.stream()
        .filter(s -> s.gpa >= 4.0)
        .sorted(Comparator.comparing(s -> s.name))
        .map(s -> s.name)

}

Optional<T> — bo'lishi yoki bo'lmasligi mumkin bo'lgan qiymat uchun konteyner. null'siz xavfsiz ishlash uchun.
