Collections Framework

Java'da ma'lumotlarni saqlash va boshqarish uchun yaratilgan kuchli kutubxona. List, Set, Map, Queue va boshqalar.

Dars maqsadi: Collections ierarxiyasi, asosiy interfaces (List, Set, Map), ularning implementatsiyalari (ArrayList, HashMap va h.k.), tezligi (Big-O), va qaysi vaziyatda qaysi birini ishlatishni o'rganamiz.

Collections Framework nima?

Collection (to'plam) — bir nechta obyektlarni bitta strukturada saqlash uchun container. Java'da java.util paketida joylashgan.

Framework — bu shunchaki kutubxona emas, balki interfaces + implementations + algoritmlar birgalikda yig'ilgan tizim.

Asoslari Java 1.2 (1998) da kiritilgan va keyin doimo takomillashtirib borilgan.

  • Interfaces — List, Set, Map, Queue...
  • Classes (implementatsiyalar) — ArrayList, HashMap...
  • Algoritmlar — Collections.sort(), Collections.shuffle()...
  • Iteratorlar — to'plamlarni aylanib chiqish vositalari

Nima uchun kerak? Array kamchiliklari

Oddiy array juda cheklangan — Collections aynan shu muammolarni hal qiladi.

O'lchamQattiq (yaratilganda belgilanadi)Dinamik (o'zgarishi mumkin)

Array'dan voz kechmaymiz — tezlik kerak bo'lganda Array, qulaylik kerak bo'lganda Collection ishlatamiz.

Collections ierarxiyasi

Iterable

Eng yuqori interface. for-each bilan aylanib chiqish imkonini beradi.

Collection

Asosiy interface — add, remove, contains, size metodlari.

Map

Alohida ierarxiya — Collection dan meros olmaydi. Key-Value juftliklarini saqlaydi.

Iyerarxiya: Iterable → Collection → (List, Set, Queue)

Map alohida turadi — chunki u juftliklar to'plami, oddiy element to'plami emas.

List interface

List — tartibli, indeksli, takror element'larga ruxsat beruvchi to'plam.

  • Har bir element indekska ega (0 dan boshlanadi)
  • Bir xil qiymat bir necha marta bo'lishi mumkin
  • Qo'shilgan tartib saqlanadi
import java.util.*;

List<String> names = new ArrayList<>();
names.add("Ali");
names.add("Vali");
names.add("Ali");      // takror — ruxsat

System.out.println(names);         // [Ali, Vali, Ali]
System.out.println(names.get(0));  // Ali
System.out.println(names.size());  // 3
List — bu interface. Uni ishlatish uchun ArrayList yoki LinkedList kabi implementatsiyani tanlaymiz.

ArrayList — dinamik massiv

Ichki strukturasi: ArrayList — bu dinamik o'zgaruvchi massiv. Ichida oddiy Object[] mavjud.

Sig'im to'lganda: yangi katta massiv yaratiladi (odatda 1.5x) va eski elementlar ko'chiriladi.

Tez ishlash: indeks bo'yicha murojaat — O(1). Lekin ortasiga qo'shish — O(n).

List<Integer> nums = new ArrayList<>();
nums.add(10);            // oxiriga
nums.add(20);
nums.add(1, 15);         // 1-indeksga
nums.set(0, 5);          // o'zgartirish
nums.remove(2);          // 2-indeks o'chirish
System.out.println(nums); // [5, 15, 20]

LinkedList — bog'langan ro'yxat

Ichki strukturasi: LinkedList — bu doubly linked list. Har bir element (Node) o'zidan oldingi va keyingi node'ga havola saqlaydi.

Qo'shish/o'chirish: ortada — O(1) (faqat havolalarni o'zgartirish).

Lekin: indeks bo'yicha murojaat — O(n) (boshidan kerakli o'ringacha yurish kerak).

LinkedList<String> list = new LinkedList<>();
list.addFirst("A");      // boshga
list.addLast("B");       // oxiriga
list.add("C");           // oxiriga
list.removeFirst();      // boshini olib tashlash
System.out.println(list); // [B, C]

// Queue/Deque sifatida ham ishlatish mumkin
list.offer("D");
list.poll();

Takrorlanuvchi elementlarni o'chirish

List ichidagi takror sonlarni olib tashlang. Tartib saqlansin.

public static List<Integer> removeDuplicates(List<Integer> list) {
    List<Integer> result = new ArrayList<>();
    for (Integer n : list) {
        if (!result.contains(n)) {
            result.add(n);
        }
    }
    return result;
}

// Yoki Set yordamida (eng oson)
public static List<Integer> removeDuplicatesSet(List<Integer> list) {
    return new ArrayList<>(new LinkedHashSet<>(list));
}
⚡ Masala 2 Easy

Eng katta va eng kichik son

List ichidan eng katta va eng kichik sonni topib, ularning yig'indisini qaytaring.

Misol
Input: [3, 7, 1, 9, 4] → 1 + 9 = 10
Input: [5] → 5 + 5 = 10 
public static int minMaxSum(List<Integer> list) {
    int min = list.get(0);
    int max = list.get(0);
    for (int n : list) {
        if (n < min) min = n;
        if (n > max) max = n;
    }
    return min + max;
}

// Yoki Collections yordamida
public static int minMaxSumShort(List<Integer> list) {
    return Collections.min(list) + Collections.max(list);
}

Misol

public static List<Integer> intersection(List<Integer> a, List<Integer> b) {

⚡ Masala 4 Hard

Eng ko'p uchragan element (Mode)

List ichida eng ko'p marta uchragan elementni toping. Bir nechta bo'lsa — ularning ro'yxatini qaytaring.

Misol
[1, 2, 2, 3, 3, 4] → [2, 3] (har biri 2 marta)
[1, 1, 1, 2, 3] → [1] (3 marta) 
public static List<Integer> findMode(List<Integer> list) {
    Map<Integer, Integer> counts = new HashMap<>();
    for (int n : list) {
        counts.put(n, counts.getOrDefault(n, 0) + 1);
    }

    int maxCount = Collections.max(counts.values());

    List<Integer> result = new ArrayList<>();
    for (Map.Entry<Integer, Integer> e : counts.entrySet()) {
        if (e.getValue() == maxCount) {
            result.add(e.getKey());
        }
    }
    Collections.sort(result);
    return result;
}

Set interface

Set — takror element'larsiz to'plam. Matematik to'plam (множество) tushunchasi.

  • Bir xil qiymat faqat 1 marta bo'lishi mumkin
  • Indeks yo'q — bevosita murojaat qilinmaydi
  • 3 ta asosiy implementatsiya: HashSet, TreeSet, LinkedHashSet
Set<String> set = new HashSet<>();
set.add("apple");
set.add("banana");
set.add("apple");     // e'tiborsiz — takror

System.out.println(set.size());           // 2
System.out.println(set.contains("apple")); // true

HashSet — Hash Table asosida

Ichki strukturasi: HashSet ichida HashMap bor. Elementlar HashMap ning kalitlari sifatida saqlanadi.

Hash funksiyasi: har bir element uchun hashCode() chaqiriladi → bucket (chelak) indeksi olinadi → o'sha chelakka qo'yiladi.

Tezligi: add, remove, contains — o'rtacha O(1)!

Tartib yo'q: elementlar qaysi tartibda chiqishi noma'lum.

Set<Integer> nums = new HashSet<>();
nums.add(5); nums.add(2); nums.add(8); nums.add(1);

System.out.println(nums); // [1, 2, 5, 8] — tartib NOTO'G'RI!
// (bu shunchaki tasodif — kafolat yo'q)

TreeSet vs LinkedHashSet

TreeSet — Red-Black Tree (muvozanatli daraxt) asosida. Elementlar doim saralangan.

LinkedHashSet — HashSet + qo'shilgan tartibni saqlovchi linked list. Tartib saqlanadi.

// TreeSet — doim sorted
Set<Integer> tree = new TreeSet<>();
tree.add(5); tree.add(2); tree.add(8); tree.add(1);
System.out.println(tree); // [1, 2, 5, 8] — saralangan

// LinkedHashSet — insertion order
Set<Integer> linked = new LinkedHashSet<>();
linked.add(5); linked.add(2); linked.add(8); linked.add(1);
System.out.println(linked); // [5, 2, 8, 1] — qo'shilgan tartib

Unique characters tekshirish

String berilgan. Barcha belgilari noyob ekanligini aniqlang (takror yo'q).

Misol
"abcdef" → true
"hello" → false (l ikki marta)
"" → true
Maslahat: Set.add() noyob bo'lsa true, takror bo'lsa false qaytaradi. 
public static boolean hasUniqueChars(String s) {
    Set<Character> seen = new HashSet<>();
    for (char c : s.toCharArray()) {
        if (!seen.add(c)) {
            return false;  // qaytarilsa — takror bor
        }
    }
    return true;
}

System.out.println(hasUniqueChars("abcdef")); // true
System.out.println(hasUniqueChars("hello"));  // false

Map interface

Map — key-value juftliklarini saqlovchi struktura. Lug'at, telefon kitobi kabi.

  • Har bir key noyob bo'lishi shart
  • Value takror bo'lishi mumkin
  • Key bo'yicha tez qidirish
Map<String, Integer> ages = new HashMap<>();
ages.put("Akobir", 22);
ages.put("Ali", 25);
ages.put("Akobir", 23);  // OVERWRITE!

System.out.println(ages.get("Akobir")); // 23
System.out.println(ages.size());        // 2
Map Collection dan meros olmaydi — bu alohida ierarxiya. Lekin keySet(), values(), entrySet() orqali Collection sifatida foydalanish mumkin.

HashMap — ichki ishlashi

HashMap ichida nima bor? Bu chelaklar (buckets) massivi. Har bir chelak — linked list yoki tree.

Qachon put(key, value) chaqiriladi:

1️⃣ key.hashCode() hisoblanadi
2️⃣ index = hash % capacity (chelak indeksi)
3️⃣ O'sha chelakka key-value juftlik qo'yiladi

Collision (to'qnashuv): ikki kalit bir xil indeksga tushsa, ular bitta chelakda saqlanadi (linked list). 8 ta'dan oshsa — TREE'ga aylanadi (Java 8+).

Tezligi: put, get, remove — o'rtacha O(1)!

Two Sum (LeetCode #1)

Massivda yig'indisi target ga teng ikkita son indekslarini toping.

Misol
nums=[2,7,11,15], target=9 → [0, 1]
nums=[3,2,4], target=6 → [1, 2]
O(n) yechim: HashMap'da har bir son uchun "kerakli juftlik"ni izlash. 
public static int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{ map.get(complement), i };
        }
        map.put(nums[i], i);
    }
    return new int[0];
}

System.out.println(Arrays.toString(twoSum(new int[]{2,7,11,15}, 9)));
// [0, 1]

Anagram tekshirish

Ikki string anagram (bir xil harflardan tuzilgan) ekanligini aniqlang.

Misol

"hello", "world" → false
"evil", "vile" → true 
public static boolean isAnagram(String a, String b) {
    if (a.length() != b.length()) return false;

    Map<Character, Integer> count = new HashMap<>();
    for (char c : a.toCharArray()) {
        count.merge(c, 1, Integer::sum);
    }
    for (char c : b.toCharArray()) {
        count.merge(c, -1, Integer::sum);
        if (count.get(c) < 0) return false;
    }
    return true;
}
⚡ Masala 4 Hard

Top K eng ko'p uchraganlar

Massivda eng ko'p marta uchragan K ta elementni toping.

Misol
nums=[1,1,1,2,2,3], k=2 → [1, 2]
nums=[1], k=1 → [1] 
public static List<Integer> topK(int[] nums, int k) {
    Map<Integer, Integer> count = new HashMap<>();
    for (int n : nums) {
        count.merge(n, 1, Integer::sum);
    }

    // PriorityQueue bilan eng ko'p uchragan elementni topish
    PriorityQueue<Integer> heap = new PriorityQueue<>(
        (a, b) -> count.get(b) - count.get(a)
    );
    heap.addAll(count.keySet());

    List<Integer> result = new ArrayList<>();
    for (int i = 0; i < k; i++) {
        result.add(heap.poll());
    }
    return result;
}

Queue va Stack

Queue (navbat) — FIFO (First In, First Out). Birinchi kelgan birinchi chiqadi.

Stack (stek) — LIFO (Last In, First Out). Oxirgi kelgan birinchi chiqadi.

// Queue (FIFO)
Queue<Integer> queue = new LinkedList<>();
queue.offer(1); queue.offer(2); queue.offer(3);
queue.poll();    // 1 (birinchi qo'shilgan)

// Deque (ikki tomonlama queue) — ko'proq ishlatiladi
Deque<Integer> deque = new ArrayDeque<>();
deque.addFirst(0);
deque.addLast(1);

// Stack (LIFO) — Deque'dan foydalanish tavsiya etiladi
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1); stack.push(2); stack.push(3);
stack.pop();     // 3 (oxirgi qo'shilgan)
Eski java.util.Stack klassi ishlatilmaydi — uning o'rniga ArrayDeque tavsiya etiladi.

Comparable interface

Comparable — obyektni "tabiiy tartibda" saralash uchun. Klass o'zi implementatsiya qiladi. 

public class Student implements Comparable<Student> {
    String name;
    int score;

    public Student(String n, int s) {
        this.name = n; this.score = s;
    }

    @Override
    public int compareTo(Student other) {
        return Integer.compare(this.score, other.score);
        // manfiy → this kichik
        // 0 → teng
        // musbat → this katta
    }
}

List<Student> students = new ArrayList<>();
students.add(new Student("Ali", 85));
students.add(new Student("Vali", 70));
students.add(new Student("Akobir", 95));

Collections.sort(students);
// Vali(70), Ali(85), Akobir(95)

Comparator interface

Comparator — alohida saralash strategiyasi. Klassni o'zgartirmasdan har xil saralash mumkin. 

List<Student> students = /* ... */;

// 1. Ism bo'yicha
students.sort(Comparator.comparing(s -> s.name));

// 2. Bal bo'yicha teskari
students.sort((a, b) -> b.score - a.score);
// yoki
students.sort(Comparator.comparingInt((Student s) -> s.score).reversed());

// 3. Avval ism, keyin bal
students.sort(
    Comparator.comparing((Student s) -> s.name)
              .thenComparingInt(s -> s.score)
);
Comparable — bitta tabiiy tartib (klass ichida). Comparator — bir nechta tartib (tashqarida).

Eslatma: Big-O — nazariy tezlik. Amalda HashMap har doim juda tez. Lekin hash collision ko'p bo'lsa, sekinlashishi mumkin.

Qaysi collection qachon?

  • ArrayList — odatda List uchun default tanlov. Indeks bilan tez ishlash kerak bo'lsa.
  • LinkedList — faqat boshidan/oxiridan tez-tez qo'shish/o'chirish kerak bo'lsa.
  • HashSet — noyob elementlar va tartib muhim emas.
  • TreeSet — noyob elementlar va doim saralangan bo'lishi kerak.
  • LinkedHashSet — noyob + qo'shilgan tartibni saqlash.
  • HashMap — key-value, tartib muhim emas. Eng ko'p ishlatiladi.
  • TreeMap — kalitlar bo'yicha sorted iteratsiya kerak.
  • ArrayDeque — Stack yoki Queue kerak bo'lsa.
Qoida: e'lon qilishda List<T> list = new ArrayList<>() — interface ko'rsating, keyin kerak bo'lsa implementatsiyani osongina almashtirasiz.
📝 Amaliy topshiriqlar Mustaqil

10 ta murakkab vazifa

Quyidagi 10 ta vazifani o'zingiz mustaqil yechib ko'ring:

  • 1. Group Anagrams. String massivini guruhlarga ajrating — har bir guruh anagram bo'lsin.
    ["eat","tea","tan","ate","nat","bat"] → [["eat","tea","ate"],["tan","nat"],["bat"]]
  • 2. Eng ko'p uchragan harf. String'da eng ko'p marta uchragan harfni va sonini qaytaring.
    "bookkeeper" → ('k', 3) yoki ('e', 3)
  • 3. Subarray sum equals K. Yig'indisi K ga teng bo'lgan subarraylar sonini toping (HashMap bilan O(n)).
    nums=[1,1,1], k=2 → 2
  • 4. LRU Cache. Eng kam ishlatilgan elementni avtomatik o'chiruvchi cache yarating (LinkedHashMap bilan).
    capacity=2, put(1,1), put(2,2), get(1), put(3,3) → 2 chiqib ketadi
  • 5. Word frequency. Matnda har bir so'zning chastotasini sanang va eng ko'p uchragan 5 tasini toping.
    "the cat and the dog" → {the=2, cat=1, and=1, dog=1}
  • 6. Intersection of two arrays. Ikki massiv kesishmasini noyob elementlar bilan toping (Set yordamida).
    [1,2,2,1] ∩ [2,2] → [2]
  • 7. Phone book. HashMap<String, List<String>> — bir kishida bir nechta telefon raqami.
    book.add("Akobir", "+99890..."), book.find("Akobir") → list
  • 8. Sort by frequency. Massivni har bir elementning chastotasi bo'yicha saralash.
    [1,1,2,2,2,3] → [2,2,2,1,1,3]
  • 9. Balanced parentheses. Stack bilan qavslar muvozanatda ekanini tekshirish.
    "({[]})" → true, "({[})" → false
  • 10. Sliding window maximum. K kenglikdagi har bir oynaning maksimumini toping (Deque bilan O(n)).
    nums=[1,3,-1,-3,5,3,6,7], k=3 → [3,3,5,5,6,7]
Har bir vazifa uchun: Big-O ni hisoblang, optimal collection tanlang, edge case'larni tekshiring.
