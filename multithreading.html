⚡ 5-Mavzu

Multithreading

Bir vaqtda bir nechta vazifani bajarish san'ati. Java'da Thread, Runnable, ExecutorService, CompletableFuture va sinxronizatsiya.

Dars maqsadi: Concurrency asoslari, Race Condition, Deadlock, Synchronization, ExecutorService va zamonaviy CompletableFuture'ni chuqur va amaliy o'rganamiz.

Process vs Thread

Process — ishlab turgan dasturning nusxasi. O'z xotirasi, fayllari, resurslari bilan.

Thread (oqim) — process ichidagi ish birligi. Bir process ichida ko'p thread bo'lishi mumkin va ular xotirani baham ko'radi.

Bir thread = bir "kod oqimi". CPU har bir thread'ni alohida bajaradi (yoki almashtirib turadi).

XususiyatProcessThread
XotiraAlohidaBirgalikda
Yaratish vaqtiSekin (og'ir)Tez (yengil)
AloqaMurakkab (IPC)Oddiy (umumiy o'zgaruvchilar)

Multithreading parallellik beradi: bir vaqtda fayl yuklash + UI yangilash + ma'lumotni qayta ishlash.

Thread Life Cycle

Thread quyidagi holatlardan (states) o'tadi:

NEW          → start() chaqirilmagan, hali ishlamayapti
  ↓
RUNNABLE     → bajarishga tayyor yoki bajarilmoqda
  ↓
BLOCKED      → boshqa thread monitor (qulf)ni ushlab turibdi
  ↓
WAITING      → wait() yoki join() — chaqirib chaqiruvni kutmoqda
  ↓
TIMED_WAITING → sleep(N) yoki wait(N) — vaqt bilan kutish
  ↓
TERMINATED   → run() metodi tugadi
Thread t = new Thread(() -> System.out.println("Salom"));
System.out.println(t.getState());  // NEW

t.start();
// RUNNABLE → TERMINATED

Thread.sleep(100);
System.out.println(t.getState());  // TERMINATED

Thread yaratish — 2 ta usul

Java'da thread yaratishning 2 ta asosiy yo'li bor:

// 1-USUL: Thread klassidan meros olish
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Salom Thread'dan!");
    }
}

MyThread t = new MyThread();
t.start();  // run() ni chaqirma — start() chaqir!

// 2-USUL: Runnable interface (TAVSIYA ETILADI)
Runnable task = () -> System.out.println("Salom Runnable'dan!");
Thread t2 = new Thread(task);
t2.start();

// Yoki bevosita
new Thread(() -> System.out.println("Hi!")).start();
Diqqat: t.run() chaqirsangiz — yangi thread ochilmaydi! Joriy thread'da bajariladi. Faqat t.start() orqali yangi thread paydo bo'ladi.
Runnable yaxshiroq — chunki Java'da multiple inheritance yo'q. Runnable ishlatsangiz, boshqa klassdan ham meros olishingiz mumkin.

Thread asosiy metodlar

Thread t = new Thread(() -> { /* ... */ });

// Boshlash
t.start();

// Joriy thread'ni vaqtga to'xtatish
Thread.sleep(1000);   // 1 sekund

// Boshqa thread tugashini kutish
t.join();             // shu joyda turamiz to'xtaguncha

// Thread'ni to'xtatishga harakat (yumshoq)
t.interrupt();

// Hozirgi thread haqida
Thread current = Thread.currentThread();
System.out.println(current.getName());     // Thread-0
System.out.println(current.getId());       // raqam
System.out.println(current.getState());    // RUNNABLE

// Daemon thread — background, asosiy thread tugagach o'ladi
t.setDaemon(true);

// Priority — JVM uchun maslahat (kafolat emas)
t.setPriority(Thread.MAX_PRIORITY);  // 10
t.setPriority(Thread.MIN_PRIORITY);  // 1
t.setPriority(Thread.NORM_PRIORITY); // 5

Massivni teng bo'lib qidirish

Katta massivda target sonni topish. N ta thread'ga teng bo'lib bering — har biri o'z qismida qidirsin. Topgan birinchi thread to'xtatsin.

Misol
arr = [1..1000000], target = 753421, threads = 4
→ Thread-3 ichida topiladi 
public static int parallelSearch(int[] arr, int target, int threadCount) throws InterruptedException {
    int[] foundIndex = {-1};
    Thread[] threads = new Thread[threadCount];
    int chunkSize = arr.length / threadCount;

    for (int t = 0; t < threadCount; t++) {
        final int start = t * chunkSize;
        final int end = (t == threadCount - 1) ? arr.length : start + chunkSize;

        threads[t] = new Thread(() -> {
            for (int i = start; i < end; i++) {
                if (foundIndex[0] != -1) return; // boshqa thread topdi
                if (arr[i] == target) {
                    foundIndex[0] = i;
                    return;
                }
            }
        });
        threads[t].start();
    }

    for (Thread t : threads) t.join();
    return foundIndex[0];
}

Ping Pong threads

Ikki thread navbatma-navbat 10 marta "PING" va "PONG" chiqarsin. Tartib qat'iy bo'lsin: PING → PONG → PING → PONG.

Misol
PING
PONG
PING
PONG... (10 marta har birida)
Maslahat: wait()/notify() yoki volatile flag ishlatish kerak. synchronized blok bilan. 
public class PingPong {
    private static final Object lock = new Object();
    private static boolean pingTurn = true;

    public static void main(String[] args) throws Exception {
        Thread ping = new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                synchronized (lock) {
                    while (!pingTurn) lock.wait();
                    System.out.println("PING");
                    pingTurn = false;
                    lock.notify();
                }
            }
        });

        Thread pong = new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                synchronized (lock) {
                    while (pingTurn) lock.wait();
                    System.out.println("PONG");
                    pingTurn = true;
                    lock.notify();
                }
            }
        });

        ping.start(); pong.start();
        ping.join();  pong.join();
    }
}

Race Condition — asosiy muammo

Bir nechta thread bir vaqtda bir o'zgaruvchini o'zgartirsa, kutilmagan natija chiqadi.

public class Counter {
    int count = 0;
    public void increment() { count++; } // tahlikali!
}

// 2 ta thread, har biri 10000 marta increment qiladi
Counter c = new Counter();
Thread t1 = new Thread(() -> { for (int i=0;i<10000;i++) c.increment(); });
Thread t2 = new Thread(() -> { for (int i=0;i<10000;i++) c.increment(); });

t1.start(); t2.start(); t1.join(); t2.join();
System.out.println(c.count); // 20000 kutamiz, lekin 15243 yoki boshqa son!

Nima uchun? count++ aslida 3 ta amaldan iborat:

1️⃣ count ni o'qish
2️⃣ 1 qo'shish
3️⃣ Yangi qiymatni yozish

Ikkala thread bir vaqtda 1-bosqichni bajarsa, bir xil qiymatni o'qishadi va bir xil natijani yozishadi — bir oshirish yo'qoladi.

synchronized va volatile

// synchronized metod — butun metod bitta thread'ga
public synchronized void increment() {
    count++;
}

// synchronized blok — faqat kerakli qismi
public void doSomething() {
    // Bu qism parallel ishlashi mumkin
    System.out.println("Boshlandi");

    synchronized (this) {
        count++;  // Faqat shu qism kritik
    }

    System.out.println("Tugadi");
}

// O'z lock obyekt — yaxshiroq
private final Object lock = new Object();
synchronized (lock) {
    // ...
}

volatile — JVM'ga aytadi: "bu o'zgaruvchini cache'lamasdan, har doim main memory'dan o'qi". Lekin atomicity bermaydi.

volatile + bir thread yozadi, ko'p thread o'qiydi → OK.

volatile ko'p thread yozadi → YO'Q, synchronized kerak!

ReentrantLock — yangi yondashuv

import java.util.concurrent.locks.*;

private final ReentrantLock lock = new ReentrantLock();

public void increment() {
    lock.lock();
    try {
        count++;
    } finally {
        lock.unlock();  // ALBATTA finally'da!
    }
}

// Afzalliklari (synchronized'dan ko'ra):
// - tryLock() — bloklamasdan urinish
// - lockInterruptibly() — interrupt'larga javob beradi
// - fair lock — birinchi kelgan birinchi ishlatadi
// - Condition'lar — kuchli wait/notify

ReentrantLock fairLock = new ReentrantLock(true); // fair

if (lock.tryLock()) {
    try { /* ... */ }
    finally { lock.unlock(); }
} else {
    System.out.println("Band, keyin urinaman");
}

Thread-safe BankAccount

BankAccount klassini xavfsiz qiling: deposit, withdraw, getBalance bir nechta thread'da chaqirilishi mumkin.

Misol
100 ta thread parallel: 10 marta deposit(1) va 10 marta withdraw(1)
Boshlang'ich balans = 1000 → Yakuniy balans = 1000 (har doim!) 
public class BankAccount {
    private double balance;
    private final Object lock = new Object();

    public BankAccount(double initial) { this.balance = initial; }

    public void deposit(double amount) {
        synchronized (lock) {
            balance += amount;
        }
    }

    public boolean withdraw(double amount) {
        synchronized (lock) {
            if (amount > balance) return false;
            balance -= amount;
            return true;
        }
    }

    public double getBalance() {
        synchronized (lock) {
            return balance;
        }
    }
}

Deadlock — o'lik qulf

Deadlock — ikki yoki ko'p thread bir-birini cheksiz kutib turadi. Hech kim ishlay olmaydi.

Klassik vaziyat: Thread-A lock1ni oldi va lock2ni kutmoqda. Thread-B aksincha — lock2ni oldi va lock1ni kutmoqda.

Object lock1 = new Object();
Object lock2 = new Object();

Thread a = new Thread(() -> {
    synchronized (lock1) {
        try { Thread.sleep(100); } catch (Exception e) {}
        synchronized (lock2) {  // kutib qolamiz — DEADLOCK
            System.out.println("A");
        }
    }
});

Thread b = new Thread(() -> {
    synchronized (lock2) {
        try { Thread.sleep(100); } catch (Exception e) {}
        synchronized (lock1) {  // kutib qolamiz — DEADLOCK
            System.out.println("B");
        }
    }
});
Oldini olish: Hamma joyda lock'larni bir xil tartibda oling! Yoki tryLock(timeout) ishlating.

wait / notify (Producer-Consumer)

Klassik pattern: bir thread ishlab chiqaradi, ikkinchisi iste'mol qiladi. 

class Queue {
    private final List<Integer> data = new ArrayList<>();
    private final int CAPACITY = 5;

    public synchronized void produce(int item) throws InterruptedException {
        while (data.size() == CAPACITY) {
            wait();  // qulf chiqarib, kutadi
        }
        data.add(item);
        System.out.println("Produced: " + item);
        notifyAll();  // consumer'larni uyg'otish
    }

    public synchronized int consume() throws InterruptedException {
        while (data.isEmpty()) {
            wait();
        }
        int item = data.remove(0);
        System.out.println("Consumed: " + item);
        notifyAll();
        return item;
    }
}
wait(), notify(), notifyAll() — faqat synchronized blok ichida chaqirilishi mumkin. wait() qulfni chiqaradi, notify() kutayotgan thread'ni uyg'otadi.

ExecutorService — zamonaviy yondashuv

Muammo: Har safar new Thread() qilish — qimmat. Operatsion tizimga murojaat qilinadi.

Yechim: Thread Pool — oldindan tayyor thread'lar to'plami. ExecutorService ularni boshqaradi.

Sizning kodingiz vazifa yuboradi, pool kim ishlashini hal qiladi.

ExecutorService executor = Executors.newFixedThreadPool(4);

// Vazifa yuborish
executor.submit(() -> System.out.println("Task 1"));
executor.submit(() -> System.out.println("Task 2"));

// Yopish — yangi vazifalar qabul qilmaydi
executor.shutdown();

// Hammasini kutish
executor.awaitTermination(1, TimeUnit.MINUTES);

// Try-with-resources (Java 19+)
try (var pool = Executors.newFixedThreadPool(4)) {
    pool.submit(...);
}  // shutdown avtomatik

CompletableFuture — zamonaviy async

Future'ning kamchiligi: get() chaqirsangiz, asosiy thread bloklanadi. Chain qilish, callback qo'shish qiyin.

CompletableFuture (Java 8+): callback'lar, chain'lar, combinatsiyalar, exception handling — barchasi funksional uslubda.

// Asincrhon vazifa boshlash
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    sleep(1000);
    return "Salom";
});

// Bloklamasdan callback
future.thenAccept(result -> System.out.println(result));

// Yoki olish (block qiladi)
String result = future.get();

// Tayyor qiymatlar
CompletableFuture<String> ready = CompletableFuture.completedFuture("Tayyor");

CompletableFuture — chain va combine

// Chain — natijani uzatib boradi
CompletableFuture
    .supplyAsync(() -> "akobir")                    // String
    .thenApply(String::toUpperCase)                  // String → String: "AKOBIR"
    .thenApply(s -> "Salom, " + s)                  // → "Salom, AKOBIR"
    .thenAccept(System.out::println);                // → chiqarish

// Combine — ikki natijani birlashtirish
CompletableFuture<Integer> a = CompletableFuture.supplyAsync(() -> 5);
CompletableFuture<Integer> b = CompletableFuture.supplyAsync(() -> 7);

CompletableFuture<Integer> sum = a.thenCombine(b, Integer::sum);
System.out.println(sum.get()); // 12

// allOf — barchasini kutish
CompletableFuture<Void> all = CompletableFuture.allOf(a, b);
all.thenRun(() -> System.out.println("Ikkalasi tayyor"));

// anyOf — birinchi tayyor bo'lganini olish
CompletableFuture<Object> first = CompletableFuture.anyOf(a, b);

// Exception handling
.exceptionally(ex -> "Default")
.handle((result, ex) -> ex != null ? "Error" : result);

Atomic va Concurrent Collections

Java thread-safe ma'lumot strukturalari taqdim etadi — qo'l bilan synchronized yozish o'rniga. 

// AtomicInteger, AtomicLong — atomic operatsiyalar
AtomicInteger counter = new AtomicInteger(0);
counter.incrementAndGet();    // ++ atomik
counter.compareAndSet(0, 5);  // CAS (Compare-And-Swap)
int value = counter.get();

// AtomicReference
AtomicReference<String> ref = new AtomicReference<>("initial");
ref.set("new value");

// ConcurrentHashMap — HashMap'ning thread-safe versiyasi
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();
map.put("a", 1);
map.compute("a", (k, v) -> v + 1);     // atomik update
map.putIfAbsent("b", 2);                // bo'lmasa qo'shish

// CopyOnWriteArrayList — kam yoziladigan, ko'p o'qiladigan
List<String> list = new CopyOnWriteArrayList<>();

// BlockingQueue — Producer-Consumer uchun ideal
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);
queue.put(1);    // to'lsa kutib turadi
int val = queue.take(); // bo'sh bo'lsa kutadi

CAS (Compare-And-Swap) — CPU darajasidagi atomik instruktsiya. "Agar qiymat X bo'lsa, Y qilib o'rnat". Lock'larsiz ishlaydi va juda tez.
