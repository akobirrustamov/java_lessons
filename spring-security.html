Spring Security

Java backend'ning xavfsizlik qalqoni. Authentication, Authorization, JWT, Roles va kuchli kriptografiya.

Dars maqsadi: Spring Security arxitekturasi, Authentication flow, JWT tuzilmasi, Authorization (URL va method-level), BCrypt va xavfsiz API yaratishni chuqur o'rganamiz.

Authentication vs Authorization

Authentication (kim?) — foydalanuvchining kimligini aniqlash. Login, parol, token tekshirish.

Authorization (nima qila oladi?) — autentifikatsiyadan o'tgan user'ga nima ruxsat berilganini tekshirish.

Tartib: avval Authentication, keyin Authorization.

AuthenticationAuthorization
SavolSen kimsan?Senga nima ruxsat?
MisolLogin va parolAdmin huquqi
HTTP code401 Unauthorized403 Forbidden
Spring classAuthenticationGrantedAuthority
Aralashtirmang: 401 = "men senga ishonmayman" (login qil), 403 = "kim ekanligingni bilaman, lekin bu joyga ruxsat yo'q".

Spring Security arxitekturasi — Filter Chain

Spring Security Servlet Filter chain ustiga qurilgan. Har bir HTTP so'rov filter'lar zanjiridan o'tadi.

Filter'lar aniq tartibda ishlaydi: SecurityContextPersistenceFilter → AuthenticationFilter → AuthorizationFilter va h.k.

HTTP Request
     ↓
┌────────────────────────────────────────────┐
│  Servlet Filter Chain                       │
│                                             │
│  1. SecurityContextPersistenceFilter        │ ← context'ni tiklash
│  2. HeaderWriterFilter                      │ ← security headers
│  3. CorsFilter                              │ ← CORS
│  4. CsrfFilter                              │ ← CSRF
│  5. LogoutFilter                            │
│  6. UsernamePasswordAuthenticationFilter    │ ← form login
│  7. JwtAuthenticationFilter (custom)        │ ← JWT
│  8. SessionManagementFilter                 │
│  9. ExceptionTranslationFilter              │ ← 401/403
│ 10. AuthorizationFilter                     │ ← qoidalar tekshirish
└──────────────────┬─────────────────────────┘
                   ↓
              DispatcherServlet → Controller
Custom JWT filter qo'shganda — odatda UsernamePasswordAuthenticationFilter dan oldin qo'yiladi.

SecurityFilterChain konfiguratsiyasi

@Configuration
@EnableWebSecurity
@EnableMethodSecurity  // @PreAuthorize uchun
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())   // API uchun (JWT bilan)
            .cors(Customizer.withDefaults())
            .sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**", "/api/public/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .requestMatchers(HttpMethod.GET, "/api/products/**").permitAll()
                .requestMatchers(HttpMethod.POST, "/api/products/**").hasAnyRole("ADMIN", "MANAGER")
                .anyRequest().authenticated()
            )
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint(authEntryPoint)  // 401
                .accessDeniedHandler(accessDeniedHandler)  // 403
            )
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
            .build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12);  // strength 12 = ~250ms hash
    }
}

UserDetailsService va UserDetails

UserDetailsService — Spring Security'ning "DB'dan user'ni qanday olish" haqidagi savolga javob beruvchi interface.

UserDetails — Spring Security tushunadigan user obyekti formati.

@Service
@RequiredArgsConstructor
public class CustomUserDetailsService implements UserDetailsService {

    private final UserRepository userRepository;

    @Override
    public UserDetails loadUserByUsername(String email)
            throws UsernameNotFoundException {
        User user = userRepository.findByEmail(email)
            .orElseThrow(() -> new UsernameNotFoundException("User not found"));

        return org.springframework.security.core.userdetails.User.builder()
            .username(user.getEmail())
            .password(user.getPasswordHash())   // hash, plain emas!
            .authorities(getAuthorities(user))
            .accountExpired(false)
            .accountLocked(!user.isActive())
            .credentialsExpired(false)
            .disabled(!user.isEnabled())
            .build();
    }

    private List<GrantedAuthority> getAuthorities(User user) {
        return user.getRoles().stream()
            .map(role -> new SimpleGrantedAuthority("ROLE_" + role.getName()))
            .toList();
    }
}

Password Encryption — BCrypt

Hech qachon parolni plain text yoki MD5/SHA1 bilan saqlash mumkin emas!

BCrypt — adaptive hash function (1999). Eng tavsiya etilgan algoritm:

1️⃣ Salt — har parol uchun avtomatik random qiymat qo'shiladi (rainbow table'larga qarshi)

2️⃣ Cost factor — hisoblash murakkabligi (default 10, har +1 — 2x sekin)

3️⃣ Self-contained — hash ichida salt va cost ham bor

PasswordEncoder encoder = new BCryptPasswordEncoder(12);

String hash = encoder.encode("mySecretPass");
// $2a$12$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
//  │   │   │
//  │   │   └── salt + hash (53 belgi)
//  │   └────── cost factor (12)
//  └────────── BCrypt algoritm versiyasi

// Tekshirish (har safar yangi hash, lekin matches() ishlaydi)
boolean matches = encoder.matches("mySecretPass", hash);  // true

// Register'da
user.setPasswordHash(encoder.encode(rawPassword));
userRepo.save(user);

// Login'da
if (!encoder.matches(rawPassword, user.getPasswordHash())) {
    throw new BadCredentialsException("Wrong password");
}
Xavfsizlik qoidalari: 1) Cost ≥ 10. 2) Parolni JSON response'da qaytarmang. 3) Login API'da timing attack'larga qarshi har doim password tekshiring (user yo'q bo'lsa ham).

public SecurityFilterChain formChain(HttpSecurity http) throws Exception {
    return http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/login", "/register").permitAll()
            .anyRequest().authenticated()
        )

            .permitAll()
        )
        .logout(logout -> logout
            .logoutSuccessUrl("/login?logout=true")
            .deleteCookies("JSESSIONID")
            .permitAll()
        )
        .build();
}

        .authorizeHttpRequests(auth -> auth.anyRequest().authenticated())

User register endpoint

POST /api/auth/register — yangi user yaratish (parol BCrypt bilan). 

@RestController
@RequestMapping("/api/auth")
@RequiredArgsConstructor
public class AuthController {
    private final UserRepository userRepo;
    private final PasswordEncoder passwordEncoder;

    @PostMapping("/register")
    @ResponseStatus(HttpStatus.CREATED)
    public UserResponse register(@Valid @RequestBody RegisterRequest req) {

        if (userRepo.existsByEmail(req.getEmail())) {
            throw new EmailExistsException();
        }

        User user = new User();
        user.setName(req.getName());
        user.setEmail(req.getEmail());
        user.setPasswordHash(passwordEncoder.encode(req.getPassword()));
        user.setEnabled(true);
        user.getRoles().add(Role.USER);  // default role

        User saved = userRepo.save(user);
        return UserResponse.from(saved);  // password HASH chiqarilmaydi!
    }
}

public class RegisterRequest {
    @NotBlank @Size(min = 2, max = 50)
    private String name;

    @NotBlank @Email
    private String email;

    @NotBlank @Size(min = 8)
    @Pattern(regexp = "^(?=.*[A-Z])(?=.*[a-z])(?=.*\\d).+$",
             message = "Katta, kichik harf va raqam bo'lishi shart")
    private String password;
}

Brute force himoyasi

Login'da 5 ta noto'g'ri urinishdan keyin akkauntni 15 daqiqaga blokirovka qilish. 

@Service
@RequiredArgsConstructor
public class LoginAttemptService {
    private final Cache<String, Integer> attemptsCache = Caffeine.newBuilder()
        .expireAfterWrite(15, TimeUnit.MINUTES)
        .maximumSize(10000)
        .build();
    private static final int MAX_ATTEMPTS = 5;

    public void loginSucceeded(String key) {
        attemptsCache.invalidate(key);
    }

    public void loginFailed(String key) {
        int attempts = attemptsCache.get(key, k -> 0) + 1;
        attemptsCache.put(key, attempts);
    }

    public boolean isBlocked(String key) {
        Integer attempts = attemptsCache.getIfPresent(key);
        return attempts != null && attempts >= MAX_ATTEMPTS;
    }
}

@PostMapping("/login")
public TokenResponse login(@RequestBody LoginRequest req, HttpServletRequest http) {
    String key = req.getEmail() + ":" + http.getRemoteAddr();

    if (loginAttemptService.isBlocked(key)) {
        throw new AccountLockedException("Try again in 15 min");
    }

    try {
        Authentication auth = authManager.authenticate(/* ... */);
        loginAttemptService.loginSucceeded(key);
        return generateTokens(auth);
    } catch (AuthenticationException e) {
        loginAttemptService.loginFailed(key);
        throw e;
    }
}

JWT — JSON Web Token tuzilmasi

JWT (RFC 7519) — JSON ma'lumotni xavfsiz uzatish standarti.

JWT 3 qismdan iborat, . bilan ajratilgan:

HEADER.PAYLOAD.SIGNATURE

Misol JWT:
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NSIsIm5hbWUiOiJBa29iaXIiLCJpYXQiOjE3MzAwMDB9.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

Decode qilingach:

HEADER (Algorithm + Type):
{
  "alg": "HS256",   // algoritm: HMAC SHA256
  "typ": "JWT"
}

PAYLOAD (Claims):
{
  "sub": "12345",         // subject (user id)
  "name": "Akobir",       // custom claim
  "iat": 1730000000,      // issued at
  "exp": 1730003600,      // expiration
  "roles": ["USER"]
}

SIGNATURE (HMAC):
HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  SECRET_KEY
)
MUHIM: Payload shifrlanmagan — faqat base64-encoded! Sirli ma'lumotni JWT'ga yozmang. Signature faqat tahrirni oldini oladi.

Access Token va Refresh Token

Muammo: Access Token uzoq amal qilsa — o'g'irlangan tokenni ishlatish oson. Qisqa bo'lsa — har 15 daqiqada login qilish kerak.

Yechim — ikkita token:

1️⃣ Access Token — qisqa muddatli (15-60 daqiqa). Har API'ga yuboriladi.

2️⃣ Refresh Token — uzoq muddatli (7-30 kun). Faqat yangi access olish uchun. Server'da DB'da saqlanadi (revoke qilish uchun).

1. LOGIN
   POST /auth/login → { accessToken, refreshToken }
   Browser saqlaydi (httpOnly cookie yoki localStorage)

2. API CALL
   GET /api/users
   Authorization: Bearer eyJ... (accessToken)
   → 200 OK

3. ACCESS TOKEN EXPIRED
   GET /api/users
   Authorization: Bearer eyJ... (eski)
   → 401 Unauthorized

4. REFRESH
   POST /auth/refresh
   { "refreshToken": "..." }
   → { "accessToken": "yangi", "refreshToken": "yangi" }

5. RETRY
   GET /api/users
   Authorization: Bearer yangi
   → 200 OK

JWT yaratish va validation

@Service
@RequiredArgsConstructor
public class JwtService {
    @Value("${app.jwt.secret}")  // 256-bit kalit
    private String secret;

    @Value("${app.jwt.access-ttl}")
    private long accessTtl;  // 3600 sekund

    public String generateAccessToken(UserDetails user) {
        return Jwts.builder()
            .setSubject(user.getUsername())
            .claim("roles", user.getAuthorities().stream()
                .map(GrantedAuthority::getAuthority).toList())
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + accessTtl * 1000))
            .signWith(Keys.hmacShaKeyFor(secret.getBytes()), SignatureAlgorithm.HS256)
            .compact();
    }

    public Claims parseToken(String token) {
        return Jwts.parserBuilder()
            .setSigningKey(Keys.hmacShaKeyFor(secret.getBytes()))
            .build()
            .parseClaimsJws(token)
            .getBody();
    }

    public boolean isValid(String token) {
        try {
            parseToken(token);
            return true;
        } catch (ExpiredJwtException | MalformedJwtException | SignatureException e) {
            return false;
        }
    }

    public String getUsername(String token) {
        return parseToken(token).getSubject();
    }
}

JWT Authentication Filter

@Component
@RequiredArgsConstructor
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    private final JwtService jwtService;
    private final UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(HttpServletRequest req,
                                   HttpServletResponse res,
                                   FilterChain chain)
            throws ServletException, IOException {

        String authHeader = req.getHeader("Authorization");

        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);

            if (jwtService.isValid(token)) {
                String username = jwtService.getUsername(token);
                UserDetails user = userDetailsService.loadUserByUsername(username);

                UsernamePasswordAuthenticationToken auth =
                    new UsernamePasswordAuthenticationToken(
                        user, null, user.getAuthorities()
                    );
                auth.setDetails(new WebAuthenticationDetailsSource().buildDetails(req));

                SecurityContextHolder.getContext().setAuthentication(auth);
            }
        }

        chain.doFilter(req, res);
    }
}

Refresh Token endpoint

POST /auth/refresh — eski refresh token bilan yangi access + refresh yaratish (token rotation). 

@Entity
public class RefreshToken {
    @Id @GeneratedValue private Long id;
    @Column(unique = true)
    private String token;
    @ManyToOne
    private User user;
    private LocalDateTime expiresAt;
    private boolean revoked;
}

@PostMapping("/refresh")
public TokenResponse refresh(@RequestBody RefreshRequest req) {
    RefreshToken old = refreshTokenRepo.findByToken(req.getRefreshToken())
        .orElseThrow(() -> new InvalidTokenException("Token not found"));

    if (old.isRevoked() || old.getExpiresAt().isBefore(LocalDateTime.now())) {
        throw new InvalidTokenException("Token expired or revoked");
    }

    // Token rotation — eski refresh'ni o'chirib, yangi berish
    old.setRevoked(true);
    refreshTokenRepo.save(old);

    User user = old.getUser();
    UserDetails userDetails = userDetailsService.loadUserByUsername(user.getEmail());

    String newAccess = jwtService.generateAccessToken(userDetails);
    String newRefresh = generateAndSaveRefreshToken(user);

    return new TokenResponse(newAccess, newRefresh, "Bearer", 3600);
}

@PostMapping("/logout")
public void logout(@RequestBody RefreshRequest req) {
    refreshTokenRepo.findByToken(req.getRefreshToken())
        .ifPresent(t -> { t.setRevoked(true); refreshTokenRepo.save(t); });
}

Authorization — URL based

@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    return http
        .authorizeHttpRequests(auth -> auth
            // Public
            .requestMatchers("/api/auth/**", "/api/public/**").permitAll()

            // HTTP metodga qarab
            .requestMatchers(HttpMethod.GET, "/api/products/**").permitAll()
            .requestMatchers(HttpMethod.POST, "/api/products/**").hasRole("ADMIN")
            .requestMatchers(HttpMethod.PUT, "/api/products/**").hasRole("ADMIN")
            .requestMatchers(HttpMethod.DELETE, "/api/products/**").hasRole("ADMIN")

            // Bir nechta role
            .requestMatchers("/api/orders/**").hasAnyRole("USER", "ADMIN")

            // Custom mantiq
            .requestMatchers("/api/admin/**").access((auth, ctx) -> {
                boolean isAdmin = auth.get().getAuthorities().stream()
                    .anyMatch(a -> a.getAuthority().equals("ROLE_ADMIN"));
                boolean fromOffice = ctx.getRequest().getRemoteAddr().startsWith("10.");
                return new AuthorizationDecision(isAdmin && fromOffice);
            })

            .anyRequest().authenticated()
        )
        .build();
}

Roles vs Authorities

Authority — eng asosiy ruxsat birligi. String formatdagi yorliq: USER_READ, ORDER_DELETE.

Role — Authority'lar to'plamining nomi. Spring'da odatda ROLE_ prefiksi bilan: ROLE_ADMIN, ROLE_USER.

hasRole('ADMIN') aslida hasAuthority('ROLE_ADMIN') bilan teng.

// 1. Klassik Role-based — oddiy
public enum Role { USER, ADMIN, MANAGER }

@PreAuthorize("hasRole('ADMIN')")  // ROLE_ prefiksi avtomatik

// 2. Authority-based — granular
public enum Permission {
    USER_READ, USER_WRITE, USER_DELETE,
    ORDER_READ, ORDER_WRITE
}

@PreAuthorize("hasAuthority('USER_DELETE')")

// 3. Aralash — eng yaxshi yondashuv
public enum Role {
    USER(Set.of(USER_READ, ORDER_READ)),
    MANAGER(Set.of(USER_READ, ORDER_READ, ORDER_WRITE)),
    ADMIN(Set.of(USER_READ, USER_WRITE, USER_DELETE, ORDER_READ, ORDER_WRITE));

    private final Set<Permission> permissions;
}

// loadUserByUsername'da:
return User.builder()
    .username(user.getEmail())
    .password(user.getPasswordHash())
    .authorities(
        user.getRoles().stream()
            .flatMap(r -> Stream.concat(
                Stream.of(new SimpleGrantedAuthority("ROLE_" + r.name())),
                r.getPermissions().stream().map(p -> new SimpleGrantedAuthority(p.name()))
            )).toList()
    ).build();

@Component("orderSecurity")
public class OrderSecurityService {
    public boolean isOwnerOrAdmin(Long orderId, Authentication auth) {
        if (auth.getAuthorities().stream()
            .anyMatch(a -> a.getAuthority().equals("ROLE_ADMIN"))) return true;

        UserDetails user = (UserDetails) auth.getPrincipal();
        return orderRepo.existsByIdAndUserEmail(orderId, user.getUsername());

// User entity'ga tenant_id qo'shamiz
@Entity
public class User {
    @Id private Long id;
    private String email;
    @Column(name = "tenant_id")
    private Long tenantId;
}

// Custom UserDetails — tenant ham bor
public class CustomUserDetails implements UserDetails {
    private final User user;
    public Long getTenantId() { return user.getTenantId(); }
}

// Hibernate filter — har query'ga avtomatik WHERE qo'shadi
@Entity
@FilterDef(name = "tenantFilter", parameters = @ParamDef(name = "tenantId", type = "long"))
@Filter(name = "tenantFilter", condition = "tenant_id = :tenantId")
public class Product { /* ... */ }

// Filter ichidagi user uchun tenant'ni o'rnatish
@Component
public class TenantFilter extends OncePerRequestFilter {
    private final EntityManager em;

    @Override
    protected void doFilterInternal(/* ... */) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth != null && auth.getPrincipal() instanceof CustomUserDetails user) {
            Session session = em.unwrap(Session.class);
            session.enableFilter("tenantFilter").setParameter("tenantId", user.getTenantId());
        }
        chain.doFilter(req, res);
    }
}

// Endi har query avtomatik tenant_id bo'yicha filtrlanadi!

Exception Handling — 401 va 403

// 401 — login kerak
@Component
@RequiredArgsConstructor
public class JwtAuthEntryPoint implements AuthenticationEntryPoint {
    private final ObjectMapper mapper;

    @Override
    public void commence(HttpServletRequest req, HttpServletResponse res,
                        AuthenticationException ex) throws IOException {
        res.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
        res.setContentType("application/json");
        mapper.writeValue(res.getWriter(), Map.of(
            "status", 401,
            "error", "Unauthorized",
            "message", "Authentication required",
            "path", req.getRequestURI()
        ));
    }
}

// 403 — ruxsat yo'q
@Component
@RequiredArgsConstructor
public class CustomAccessDeniedHandler implements AccessDeniedHandler {
    private final ObjectMapper mapper;

    @Override
    public void handle(HttpServletRequest req, HttpServletResponse res,
                      AccessDeniedException ex) throws IOException {
        res.setStatus(HttpServletResponse.SC_FORBIDDEN);
        res.setContentType("application/json");
        mapper.writeValue(res.getWriter(), Map.of(
            "status", 403,
            "error", "Forbidden",
            "message", "You don't have permission"
        ));
    }
}

// SecurityConfig'ga ulash
.exceptionHandling(ex -> ex
    .authenticationEntryPoint(jwtAuthEntryPoint)
    .accessDeniedHandler(customAccessDeniedHandler))

CSRF va CORS

CSRF (Cross-Site Request Forgery) — boshqa sayt sizning sayt nomidan request yuborishi.

Himoya: Token, har request'da yuborilishi kerak. Spring Security default'da yoqilgan.

Stateless API (JWT) uchun — odatda csrf.disable() qilamiz (Bearer header CSRF'ga immune).

// API (stateless) uchun CSRF'ni o'chiramiz
http.csrf(csrf -> csrf.disable());

// Form-based ilova uchun yoqamiz (default)
http.csrf(csrf -> csrf
    .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
);

// CORS konfiguratsiyasi
@Bean
public CorsConfigurationSource corsConfig() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowedOrigins(List.of("https://frontend.com"));
    config.setAllowedMethods(List.of("GET", "POST", "PUT", "PATCH", "DELETE"));
    config.setAllowedHeaders(List.of("*"));
    config.setExposedHeaders(List.of("Authorization", "X-Total-Count"));
    config.setAllowCredentials(true);
    config.setMaxAge(3600L);

    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", config);
    return source;
}

// SecurityConfig'da:
http.cors(cors -> cors.configurationSource(corsConfig()));

OAuth2 va OpenID Connect

OAuth2 (RFC 6749) — uchinchi taraf orqali autentifikatsiya standarti. "Google bilan kirish" — bu OAuth2.

OpenID Connect (OIDC) — OAuth2 ustiga qurilgan identifikatsiya qatlami.

Spring Security ikkalasini ham qo'llab-quvvatlaydi: spring-boot-starter-oauth2-client va oauth2-resource-server.

// pom.xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>

// application.yml
spring:
  security:
    oauth2:
      client:
        registration:
          google:
            client-id: ${GOOGLE_CLIENT_ID}
            client-secret: ${GOOGLE_CLIENT_SECRET}
            scope: openid, profile, email
          github:
            client-id: ${GITHUB_CLIENT_ID}
            client-secret: ${GITHUB_CLIENT_SECRET}

// SecurityConfig
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    return http
        .authorizeHttpRequests(auth -> auth.anyRequest().authenticated())
        .oauth2Login(oauth2 -> oauth2
            .defaultSuccessUrl("/dashboard")
            .userInfoEndpoint(u -> u.userService(customOAuth2UserService))
        )
        .build();
}

// Endpoint'lar avtomatik yaratiladi:
// /oauth2/authorization/google
// /login/oauth2/code/google (callback)

10 ta murakkab vazifa

Quyidagi vazifalarni Spring Security bilan bajaring:

  • 1. To'liq Auth Service. Register, Login, Refresh, Logout, Forgot Password, Change Password — JWT bilan.
  • 2. Email verification. Register → email token jo'natish → click → activate. Token expire bilan.
  • 3. 2FA (TOTP). Google Authenticator bilan ikki bosqichli autentifikatsiya.
  • 4. OAuth2 + custom mapping. Google login, lekin DB'da local user yaratish (CustomOAuth2UserService).
  • 5. Rate Limiting. Per-IP va per-user — Bucket4j yoki Redis bilan.
  • 6. Audit Logging. Har login attempt, password change, sensitive action'larni alohida jadval'ga yozish.
  • 7. Multi-tenancy. Hibernate Filter + JWT'da tenant ID + har query'ga avtomatik filter.
  • 8. Session management. Aktiv sessiyalarni ko'rsatish va boshqalarini majburiy chiqarish ("Sign out everywhere else").
  • 9. Password policy. Murakkablik, oldingi 5 parol bilan moslik tekshirish, har 90 kun majburiy o'zgartirish.
  • 10. API Keys. Server-to-server uchun JWT emas, balki API key (DB'da hashed bo'lib saqlanadi).
Har bir vazifa uchun: OWASP Top 10 tahdidlarini hisobga oling, HTTPS only, parol va token'larni log'larda yoki response'da chiqarmang.
