OOP

Object-Oriented Programming — obyektga yo'naltirilgan dasturlash. Real dunyo obyektlarini kodda tasvirlash usuli.

Dars maqsadi: Class va Object, Constructor, Encapsulation, Inheritance, Polymorphism, Abstraction, Interface va SOLID tamoyillarini real misollar bilan o'rganib chiqamiz.

OOP nima?

OOP — kodni real dunyo obyektlari ko'rinishida tashkil etish usuli. Bu paradigma katta loyihalarni boshqarish va kengaytirishni osonlashtiradi.

  • Kodni qayta foydalanish oson (reusability)
  • Kengaytirish oson — yangi xususiyatlar qo'shish
  • Kodni boshqarish va saqlash oson (maintainability)
  • Real dunyo modeli bilan moslashadi — odamlar uchun tushunarli
  • Spring, Android, Enterprise — barcha katta Java loyihalari OOP da yoziladi
OOP 4 ta asosiy ustunga tayanadi: Encapsulation, Inheritance, Polymorphism, Abstraction.

Class, Object, Instance

Class

Sinf — obyekt yaratish uchun shablon (chizmasi). U xususiyatlar va xatti-harakatlarni belgilaydi.

Object

Obyekt — class dan yaratilgan real "narsa". Xotirada joy egallaydi va o'z qiymatlariga ega.

Instance

Nusxa — class ning konkret nusxasi. "Object" va "instance" amalda sinonim sifatida ishlatiladi.

Analogiya: Class — uy chizmasi, Object — chizma asosida qurilgan haqiqiy uy. Bitta chizmadan ko'p uy qurish mumkin.

Birinchi class — Car

Class class kalit so'zi bilan e'lon qilinadi. Obyekt new orqali yaratiladi.

public class Car {
    String brand;
    String color;
    int year;
}

// Foydalanish
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.brand = "Toyota";
        myCar.color = "Qora";
        myCar.year = 2024;

        System.out.println(myCar.brand + " " + myCar.year);
        // Toyota 2024
    }
}

Fields — atributlar

Class ichidagi o'zgaruvchilar field (atribut) deyiladi. Har bir obyekt o'z field qiymatlariga ega bo'ladi.

  • Default qiymatlar: int = 0, double = 0.0, boolean = false, String = null
  • Access modifierlar: public, private, protected, default
  • Field'lar class scope'da yashaydi — barcha metodlar ko'ra oladi
public class Student {
    public String name;        // hamma ko'radi
    private int age;           // faqat class ichida
    protected double gpa;      // shu paket + meros oluvchilar
    String university;         // default — faqat shu paket
}

Constructor

Constructor — obyekt yaratilganda avtomatik chaqiriladigan maxsus metod. Field'larga boshlang'ich qiymat berish uchun ishlatiladi. 

public class Person {
    String name;
    int age;

    // Default constructor (parametrsiz)
    public Person() {
        this.name = "Noma'lum";
        this.age = 0;
    }

    // Parameterized constructor
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// Foydalanish
Person p1 = new Person();              // Default
Person p2 = new Person("Akobir", 22);  // Parameterized
Constructor class nomi bilan bir xil va return turi yo'q (hatto void ham). Bir nechta constructor — bu constructor overloading.

this kalit so'zi

this — joriy obyektga ishora qiluvchi havola. Asosan field va parametr nomlari mos kelganda kerak bo'ladi. 

public class Book {
    String title;
    String author;

    public Book(String title, String author) {
        this.title = title;     // this.title — class field
        this.author = author;   // title — parametr
    }

    public void info() {
        System.out.println(this.title + " by " + this.author);
    }
}

Book b = new Book("Java", "Akobir");
b.info(); // Java by Akobir

Methods — metodlar

Method — class ichidagi xatti-harakat. Obyekt nima qila olishini belgilaydi. 

public class Calculator {
    // Instance method (obyektga tegishli)
    public int add(int a, int b) {
        return a + b;
    }

    // Static method (class ga tegishli)
    public static int multiply(int a, int b) {
        return a * b;
    }
}

// Foydalanish
Calculator calc = new Calculator();
System.out.println(calc.add(2, 3));         // Instance: 5
System.out.println(Calculator.multiply(4, 5)); // Static: 20
Instance method — obyekt orqali chaqiriladi (obj.method()). Static method — class orqali (ClassName.method()).

Encapsulation — ma'lumotni himoyalash

Encapsulation — OOP ning birinchi ustuni. Field'larni private qilib, ularga faqat getter va setter orqali ruxsat beriladi. 

public class Account {
    private double balance;  // tashqi kod ko'rmaydi

    public double getBalance() {
        return balance;
    }

    public void setBalance(double balance) {
        if (balance >= 0) {        // validatsiya!
            this.balance = balance;
        }
    }
}

Account acc = new Account();
acc.setBalance(1000);           // OK
acc.setBalance(-50);            // E'tiborsiz qoldiriladi
System.out.println(acc.getBalance()); // 1000
Foyda: Ma'lumot xavfsiz, validatsiya qilinadi, kelajakda kodni o'zgartirish oson.

Bank Account (Encapsulation)

BankAccount klassini yarating. balance field private bo'lsin. Ikkita metod yozing:

  • deposit(amount) — pul qo'shish (faqat musbat son)
  • withdraw(amount) — pul yechish (faqat balansda yetarli bo'lsa)
Misol
acc.deposit(1000) → balance = 1000
acc.deposit(-50) → e'tiborsiz
acc.withdraw(300) → balance = 700
acc.withdraw(2000) → e'tiborsiz (balans yetmaydi) 
public class BankAccount {
    private double balance;

    public double getBalance() {
        return balance;
    }

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public boolean withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }
}

// Test
BankAccount acc = new BankAccount();
acc.deposit(1000);
acc.withdraw(300);
System.out.println(acc.getBalance()); // 700

Counter (chegaralar bilan)

Counter klassi yarating. Boshlang'ich qiymat, minimal va maksimal chegara bilan ishlasin.

  • increment() — 1 ga oshiradi (max'dan oshmaydi)
  • decrement() — 1 ga kamaytiradi (min'dan kichik bo'lmaydi)
  • reset() — boshlang'ich qiymatga qaytaradi
Misol
new Counter(0, 5, 0) → value = 0
increment() × 7 → value = 5 (chegaraga yetdi)
decrement() × 3 → value = 2
reset() → value = 0 
public class Counter {
    private int value;
    private final int min;
    private final int max;
    private final int initial;

    public Counter(int min, int max, int initial) {
        this.min = min;
        this.max = max;
        this.initial = initial;
        this.value = initial;
    }

    public void increment() {
        if (value < max) value++;
    }

    public void decrement() {
        if (value > min) value--;
    }

    public void reset() {
        value = initial;
    }

    public int getValue() {
        return value;
    }
}

Vault — parolli omborxona

Vault klassi parol bilan himoyalanadi. 3 ta noto'g'ri urinishdan keyin omborxona bloklanadi.

  • unlock(password) — to'g'ri bo'lsa true
  • 3 ta xato → bloklash (boshqa hech qachon ochilmaydi)
  • To'g'ri parol — urinishlar hisoblagichi nolga qaytsin
Misol
v = new Vault("1234")
v.unlock("0000") → false (urinish 1)
v.unlock("1111") → false (urinish 2)
v.unlock("1234") → true (urinishlar reset)
v.unlock("a") → false (1)
v.unlock("b") → false (2)
v.unlock("c") → false (3 — BLOKLANDI) 
public class Vault {
    private final String password;
    private int attempts = 0;
    private boolean locked = false;
    private static final int MAX_ATTEMPTS = 3;

    public Vault(String password) {
        this.password = password;
    }

    public boolean unlock(String input) {
        if (locked) return false;

        if (password.equals(input)) {
            attempts = 0;
            return true;
        }

        attempts++;
        if (attempts >= MAX_ATTEMPTS) {
            locked = true;
        }
        return false;
    }

    public boolean isLocked() {
        return locked;
    }
}

super kalit so'zi

super — parent class'ga ishora. Constructor'da yoki parent metodga murojaat qilishda ishlatiladi. 

public class Person {
    String name;

    public Person(String name) {
        this.name = name;
    }

    public void greet() {
        System.out.println("Salom, men " + name);
    }
}

Method Overriding

Overriding — child class parent class metodini qayta yozadi. Bir xil signature kerak. 

public class Shape {
    public double area() {
        return 0;
    }
}

public class Circle extends Shape {
    double radius;

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}

public class Square extends Shape {
    double side;

    @Override
    public double area() {
        return side * side;
    }
}
@Override annotatsiyasi — kompilyator bizning override qilayotganimizni tekshiradi. Xato bo'lsa, ogohlantiradi (masalan, parent metod nomi xato bo'lsa).
⚡ Masala Medium

Shape iyerarxiyasi

Abstract Shape klassidan Circle, Rectangle, Triangle hosil qiling. Har biri o'z area() va perimeter() metodini implementatsiya qilsin.

Misol
Circle(5) → area = 78.54, perimeter = 31.42
Rectangle(4, 6) → area = 24, perimeter = 20
Triangle(3, 4, 5) → area = 6, perimeter = 12
Maslahat: Uchburchak yuzasi Geron formulasi orqali: S = √(p(p-a)(p-b)(p-c)), p — yarim perimetr. 
public class Shape {
    public double area() { return 0; }
    public double perimeter() { return 0; }
}

public class Circle extends Shape {
    double r;
    public Circle(double r) { this.r = r; }

    @Override public double area() { return Math.PI * r * r; }
    @Override public double perimeter() { return 2 * Math.PI * r; }
}

public class Rectangle extends Shape {
    double w, h;
    public Rectangle(double w, double h) { this.w = w; this.h = h; }

    @Override public double area() { return w * h; }
    @Override public double perimeter() { return 2 * (w + h); }
}

public class Triangle extends Shape {
    double a, b, c;
    public Triangle(double a, double b, double c) {
        this.a = a; this.b = b; this.c = c;
    }

    @Override public double area() {
        double p = perimeter() / 2;
        return Math.sqrt(p * (p-a) * (p-b) * (p-c));
    }
    @Override public double perimeter() { return a + b + c; }
}

Polymorphism — "bir necha shaklda"

Polymorphism — OOP ning uchinchi ustuni. Bitta interface, lekin turli xil amalga oshirish.

  • Runtime polymorphism — @Override orqali (asosiy)
  • Compile-time polymorphism — overloading (bir xil nomda, turli parametrlar)
  • Parent turida child obyekt — upcasting
Shape[] shapes = {
    new Circle(5),
    new Rectangle(4, 6),
    new Triangle(3, 4, 5)
};

for (Shape s : shapes) {
    System.out.println(s.area());
    // 78.54
    // 24.0
    // 6.0
}
Bu yerda s.area() — har bir obyekt o'zining area() metodini chaqiradi (runtime'da aniqlanadi). Bu polymorphism kuchidir.

instanceof va casting

instanceof — obyekt qaysi class turiga tegishliligini tekshiradi. 

Shape s = new Circle(5);

if (s instanceof Circle) {
    Circle c = (Circle) s;  // downcasting
    System.out.println("Radius: " + c.r);
}

// Java 16+ qisqaroq versiyasi (pattern matching)
if (s instanceof Circle c) {
    System.out.println("Radius: " + c.r);
}
Upcasting (child → parent) — har doim xavfsiz. Downcasting (parent → child) — xavfli, oldindan instanceof bilan tekshiring.

Payment Processor

Turli to'lov turlari uchun yagona protsessor yarating. Har bir tur komissiya oladi:

  • CashPayment: 0%
  • CardPayment: 1.5%
  • CryptoPayment: 0.5% (lekin minimal $1)
Misol
Cash(100).process() → 100
Card(100).process() → 98.5
Crypto(100).process() → 99.5
Crypto(50).process() → 49 (minimal $1 komissiya) 
public abstract class Payment {
    protected double amount;
    public Payment(double amount) { this.amount = amount; }

    public abstract double fee();

    public double process() {
        return amount - fee();
    }
}

public class CashPayment extends Payment {
    public CashPayment(double a) { super(a); }
    @Override public double fee() { return 0; }
}

public class CardPayment extends Payment {
    public CardPayment(double a) { super(a); }
    @Override public double fee() { return amount * 0.015; }
}

public class CryptoPayment extends Payment {
    public CryptoPayment(double a) { super(a); }
    @Override public double fee() {
        return Math.max(amount * 0.005, 1);
    }
}

Comparable Items

import java.util.*;

public class Item implements Comparable<Item> {
    String name;
    double price;

    public Item(String name, double price) {
        this.name = name;
        this.price = price;

    public int compareTo(Item other) {
        return Double.compare(this.price, other.price);
    }

    @Override
    public String toString() {
        return name + "(" + price + ")";
    }
}

Discount Strategy

DiscountStrategy interface'i. 3 ta turli strategiya:

  • NoDiscount: 0%
  • PercentDiscount(p): p% chegirma
  • FixedDiscount(amount): aniq summa chegirma
Misol
100 + NoDiscount → 100
100 + PercentDiscount(20) → 80
100 + FixedDiscount(15) → 85
Maslahat: Bu Strategy pattern — algoritmni runtime'da almashtirish mumkin. 
public interface DiscountStrategy {
    double apply(double price);
}

public class NoDiscount implements DiscountStrategy {
    @Override
    public double apply(double price) { return price; }
}

public class PercentDiscount implements DiscountStrategy {
    private double percent;
    public PercentDiscount(double p) { this.percent = p; }

    @Override
    public double apply(double price) {
        return price * (1 - percent / 100);
    }
}

public class FixedDiscount implements DiscountStrategy {
    private double amount;
    public FixedDiscount(double a) { this.amount = a; }

    @Override
    public double apply(double price) {
        return Math.max(0, price - amount);
    }
}

// Foydalanish
DiscountStrategy strategy = new PercentDiscount(20);
double finalPrice = strategy.apply(100); // 80

SOLID — toza kod tamoyillari

SOLID — Robert Martin (Uncle Bob) tomonidan ishlab chiqilgan 5 ta OOP tamoyili. Toza, kengaytiriladigan kod yozish uchun.

  • SRP — Single Responsibility Principle (Bitta mas'uliyat)
  • OCP — Open/Closed Principle (Ochiq/Yopiq)
  • LSP — Liskov Substitution Principle (Liskov almashinishi)
  • ISP — Interface Segregation Principle (Interface'larni ajratish)
  • DIP — Dependency Inversion Principle (Bog'liqlikni teskarisi)
SOLID — qoidalar emas, yo'l-yo'riqlar. Spring Framework butunlay SOLID asosida tuzilgan.

SRP & OCP

SRP — Single Responsibility: Har bir class faqat bitta sababga ko'ra o'zgarishi kerak. Bitta vazifani bajarsin.

// YOMON: bir class 2 ta vazifa qiladi
class User {
    void save() { /* DB'ga yozish */ }
    void sendEmail() { /* email yuborish */ }
}

// YAXSHI: har biri o'z vazifasi
class User { /* ma'lumotlar */ }
class UserRepository { void save(User u) {} }
class EmailService { void send(User u) {} }

OCP — Open/Closed: Class kengayishga ochiq, lekin o'zgartirishga yopiq bo'lsin. Yangi xususiyat qo'shganda eski kodni o'zgartirmaslik kerak.

// Discount qo'shganda Shape klassini o'zgartirmaymiz —
// yangi DiscountStrategy implementatsiyasi qo'shamiz

LSP, ISP, DIP

LSP — Liskov Substitution: Child obyekt parent o'rnida ishlatilganda kod buzilmasligi kerak.

// YOMON
class Bird { void fly() {} }
class Ostrich extends Bird { /* tuyaqush ucha olmaydi! */ }

// YAXSHI: turlarga bo'lib chiqamiz
interface FlyingBird { void fly(); }
class Sparrow implements FlyingBird {}
class Ostrich {} // alohida

ISP — Interface Segregation: Katta interface'larni kichik, maxsus interface'larga bo'ling.

DIP — Dependency Inversion: Konkret klasslarga emas, abstraksiyalarga bog'laning.

// YOMON
class Service { MySqlDb db = new MySqlDb(); }

// YAXSHI
class Service {
    Database db;
    Service(Database db) { this.db = db; } // any DB
}
📝 Amaliy topshiriqlar Mustaqil

10 ta murakkab vazifa

Quyidagi 10 ta vazifani o'zingiz mustaqil yechib ko'ring. Har biri bir nechta OOP tushunchasini birlashtiradi:

  • 1. Library tizimi. Book, Library klasslari. Kitob qo'shish, qidirish, o'chirish.
    lib.add(book) → true   |   lib.findByTitle("Java") → Book
  • 2. Hotel reservatsiya. Room, Reservation, Hotel. Xona band qilish, bo'shatish.
    hotel.book(roomNo, date) → Reservation
  • 3. Bank account ierarxiyasi. Account abstract, SavingsAccount (foiz), CheckingAccount (overdraft).
    savings.addInterest() → balance += balance * 0.05
  • 4. Employee payroll. Employee, Manager, Developer, Intern. Har birining calculatePay().
    manager.calculatePay() → baseSalary + bonus
  • 5. Online store cart. Product, CartItem, ShoppingCart. Qo'shish, o'chirish, total hisoblash.
    cart.getTotal() → sum of (qty * price)
  • 6. Animal kingdom. Animal abstract → Mammal, Bird, Fish. Har biri o'z xususiyatlari bilan.
    Interface'lar: CanSwim, CanFly, CanRun
  • 7. Vehicle fleet. Vehicle abstract → Car, Truck, Motorcycle. Yoqilg'i sarfi va tezlik farqi.
    fleet.totalFuelConsumption() → ...
  • 8. School management. Person → Student, Teacher. School klassi.
    student.gpa, teacher.subject, school.addPerson(p)
  • 9. Game character. Character abstract → Warrior, Mage, Archer. attack() har xil.
    Interface'lar: Healable, Defendable
  • 10. Notification system. Notifier interface → EmailNotifier, SmsNotifier, PushNotifier. Strategy pattern.
    service.notify(user, message)
Har bir vazifa uchun: SOLID tamoyillariga rioya qiling, encapsulation, inheritance va polymorphism'dan keng foydalaning.
