REST API

Zamonaviy backend va frontend o'rtasidagi standart muloqot tili. HTTP'ga asoslangan, resurs-yo'naltirilgan arxitektura.

Dars maqsadi: REST tamoyillari, HTTP metodlar va status code'lar, DTO pattern, Request/Response boshqaruvi, pagination, versioning, CORS va Swagger/OpenAPI'ni chuqur o'rganamiz.

REST nima?

REST (Representational State Transfer) — 2000-yilda Roy Fielding tomonidan PhD dissertatsiyasida tavsiflangan arxitektura uslubi.

REST — bu protocol emas, balki arxitektura tamoyillari to'plami. HTTP'ga asoslangan, lekin majburiy emas.

Asosiy g'oya: Hamma narsa — resurs (resource). Har bir resursning noyob URI'si bor. HTTP metodlar bilan resurslar ustida amal bajariladi.

Misol: GET /users/42 — 42-id li foydalanuvchini olish. DELETE /users/42 — uni o'chirish.

Frontend (Browser, Mobile)         Backend (Server)
        │                                  │
        │  GET /api/users/42               │
        │ ───────────────────────────────► │
        │                                  │ → DB so'rovi
        │                                  │
        │  200 OK + { "id": 42, ... }      │
        │ ◄─────────────────────────────── │
        │                                  │

REST 6 ta tamoyili (Fielding)

Roy Fielding 6 ta constraint (cheklov) belgilagan. Hammasiga rioya qilingan API — "RESTful":

  1. Client-Server — frontend va backend mustaqil rivojlanadi
  2. Stateless — har bir so'rov o'zicha to'liq. Server session saqlamaydi
  3. Cacheable — javoblarni cache qilish mumkin (HTTP cache headers)
  4. Uniform Interface — yagona, oldindan ma'lum interface (URI + metod)
  5. Layered System — qatlamli arxitektura (proxy, load balancer, gateway)
  6. Code on Demand (ixtiyoriy) — server kod yuborishi mumkin (JS)
Stateless — eng muhimi! Har bir request o'zining authentication, ma'lumot va kontekstini olib kelishi kerak. Bu ko'lamlashni osonlashtiradi (1 server vs 100 server farqi yo'q).

Safe va Idempotent — muhim tushunchalar

Safe — metod ma'lumotni o'zgartirmaydi (faqat o'qiydi). Browser oldindan cache qila oladi.

Idempotent — metodni bir necha marta chaqirilsa ham natija bir xil. Tarmoq xatosi bo'lsa, xavfsiz retry qilinadi.

MetodSafeIdempotentIzoh
GET✅✅Faqat o'qiydi
POST❌❌Har safar yangi resurs
PUT❌✅Bir xil body — bir xil natija
PATCH❌❌Kontekstga qarab
DELETE❌✅Ikkinchi marta — already deleted
Amaliy ma'no: Mobil ilova GET so'rovi bo'yicha timeout bo'lsa — xavfsiz qayta urinish mumkin. POST bilan — ehtiyot bo'ling (ikki marta to'lov!).

To'liq CRUD Controller

UserController uchun 5 ta endpoint yarating: GET (list), GET (one), POST, PUT, DELETE. 

@RestController
@RequestMapping("/api/v1/users")
public class UserController {
    private final UserService service;

    public UserController(UserService service) {
        this.service = service;
    }

    @GetMapping
    public List<User> list() {
        return service.findAll();
    }

    @GetMapping("/{id}")
    public User get(@PathVariable Long id) {
        return service.findById(id);
    }

    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public User create(@Valid @RequestBody UserRequest req) {
        return service.create(req);
    }

    @PutMapping("/{id}")
    public User update(@PathVariable Long id, @Valid @RequestBody UserRequest req) {
        return service.update(id, req);
    }

    @DeleteMapping("/{id}")
    @ResponseStatus(HttpStatus.NO_CONTENT)
    public void delete(@PathVariable Long id) {
        service.delete(id);
    }
}

PATCH — qisman yangilash

PATCH /users/{id} — faqat berilgan field'larni yangilasin. Boshqalari o'zgarmasin. 

// Request: PATCH /users/42  { "email": "new@mail.com" }
// Faqat email yangilanadi, name va boshqalari o'z holicha qoladi.

public class UserPatchRequest {
    private String name;     // optional
    private String email;    // optional
    private Integer age;     // optional (Integer — null bo'lishi mumkin)
    // getter/setter
}

@PatchMapping("/{id}")
public User patch(@PathVariable Long id, @RequestBody UserPatchRequest req) {
    User user = service.findById(id);

    if (req.getName() != null) user.setName(req.getName());
    if (req.getEmail() != null) user.setEmail(req.getEmail());
    if (req.getAge() != null) user.setAge(req.getAge());

    return service.save(user);
}

// Yoki MapStruct/ModelMapper bilan
@PatchMapping("/{id}")
public User patchV2(@PathVariable Long id, @RequestBody Map<String, Object> updates) {
    User user = service.findById(id);
    updates.forEach((key, value) -> {
        Field field = ReflectionUtils.findField(User.class, key);
        if (field != null) {
            field.setAccessible(true);
            ReflectionUtils.setField(field, user, value);
        }
    });
    return service.save(user);
}

Idempotency Key bilan POST

To'lov POST'i tarmoq xatosida ikki marta yuborilishi mumkin. Idempotency-Key header bilan ikki marta to'lovning oldini oling. 

@RestController
public class PaymentController {

    private final Map<String, Payment> cache = new ConcurrentHashMap<>();
    private final PaymentService service;

    @PostMapping("/payments")
    @ResponseStatus(HttpStatus.CREATED)
    public Payment create(
            @RequestHeader("Idempotency-Key") String key,
            @Valid @RequestBody PaymentRequest req) {

        // Avval cache'da bormi?
        Payment existing = cache.get(key);
        if (existing != null) {
            return existing;  // ikkinchi marta — eski natija
        }

        // Yangi to'lov
        Payment payment = service.process(req);
        cache.put(key, payment);
        return payment;
    }
}

// Client har safar UUID generate qiladi va o'zining yagona kalit
// sifatida yuboradi. Tarmoq xatosida xuddi shu kalit bilan retry qiladi.
// Server bir martagina to'laydi.

DTO Pattern — nima va nima uchun?

DTO (Data Transfer Object) — qatlamlar (layer) o'rtasida ma'lumot uzatish uchun maxsus obyekt.

Nima uchun kerak?

1️⃣ Xavfsizlik — parol kabi maxfiy field'lar tashqariga chiqmaydi

2️⃣ API stability — DB schema o'zgarsa, API o'zgarmaydi

3️⃣ Validation — har endpoint o'z talablari bo'yicha

4️⃣ Performance — kerakli field'lar olinadi, ortiqcha emas

┌──────────────┐   DTO    ┌───────────┐   Entity   ┌────────┐
│  Frontend    │ ◄──────► │ Controller│ ◄────────► │   DB   │
└──────────────┘  (JSON)  └───────────┘            └────────┘
                              │
                              ↓
                          UserDTO
                          ├── id
                          ├── name
                          ├── email
                          └── (password yo'q!)

                          UserEntity (DB'da)
                          ├── id
                          ├── name
                          ├── email
                          ├── password
                          ├── createdAt
                          └── deleted

Entity vs DTO — kod misol

// Entity — DB'ga moslangan
@Entity
@Table(name = "users")
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;
    private String email;
    private String passwordHash;     // tashqariga chiqmasin!
    private LocalDateTime createdAt;
    private boolean deleted;

    @OneToMany(mappedBy = "user")
    private List<Order> orders;
}
// Response DTO — frontend uchun chiqadi
public class UserResponse {
    private Long id;
    private String name;
    private String email;
    private LocalDateTime createdAt;
    // password YO'Q!
}

// Request DTO — frontend yuboradi
public class UserRequest {
    @NotBlank @Size(min = 2, max = 50)
    private String name;

    @NotBlank @Email
    private String email;

    @NotBlank @Size(min = 8)
    private String password;
    // id va createdAt — server o'zi qo'yadi
}

// Mapping
public UserResponse toResponse(User user) {
    UserResponse r = new UserResponse();
    r.setId(user.getId());
    r.setName(user.getName());
    r.setEmail(user.getEmail());
    r.setCreatedAt(user.getCreatedAt());
    return r;
}

MapStruct bilan auto-mapping

MapStruct — compile-time'da DTO-Entity mapping kodini generate qiladi. ModelMapper'dan tez va xavfsiz. 

@Mapper(componentModel = "spring")
public interface UserMapper {

    UserResponse toResponse(User user);

    @Mapping(target = "id", ignore = true)
    @Mapping(target = "createdAt", ignore = true)
    @Mapping(target = "passwordHash", source = "password")
    User toEntity(UserRequest req);

    List<UserResponse> toResponseList(List<User> users);

    // PATCH — faqat null bo'lmagan field'larni nusxalash
    @BeanMapping(nullValuePropertyMappingStrategy = NullValuePropertyMappingStrategy.IGNORE)
    void updateFromRequest(UserPatchRequest req, @MappingTarget User entity);
}

// Spring'da Bean sifatida ishlatish
@Service
public class UserService {
    private final UserMapper mapper;
    private final UserRepository repo;

    public UserResponse get(Long id) {
        User user = repo.findById(id).orElseThrow();
        return mapper.toResponse(user);
    }

    public UserResponse create(UserRequest req) {
        User entity = mapper.toEntity(req);
        return mapper.toResponse(repo.save(entity));
    }
}

ResponseEntity — Response boshqaruvi

@RestController
public class ResponseDemo {

    // 1. Oddiy qaytarish — body + 200 OK
    @GetMapping("/simple")
    public User simple() {
        return new User(1L, "Akobir");
    }

    // 2. ResponseEntity bilan — to'liq nazorat
    @PostMapping("/users")
    public ResponseEntity<User> create(@RequestBody UserRequest req) {
        User created = service.create(req);

        URI location = ServletUriComponentsBuilder
            .fromCurrentRequest()
            .path("/{id}")
            .buildAndExpand(created.getId())
            .toUri();

        return ResponseEntity
            .status(HttpStatus.CREATED)
            .header("X-Created-By", "system")
            .location(location)  // Location: /users/42
            .body(created);
    }

    // 3. Bo'sh javob
    @DeleteMapping("/users/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        service.delete(id);
        return ResponseEntity.noContent().build();  // 204 No Content
    }

    // 4. Conditional response
    @GetMapping("/users/{id}")
    public ResponseEntity<User> get(@PathVariable Long id) {
        return service.findById(id)
            .map(ResponseEntity::ok)
            .orElseGet(() -> ResponseEntity.notFound().build());
    }
}

File Upload (Multipart)

POST /upload orqali fayl yuklash. Fayl kattaligi va turi tekshirilsin. 

@PostMapping(value = "/upload", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public Map<String, String> upload(
        @RequestParam("file") MultipartFile file,
        @RequestParam(required = false) String description) {

    if (file.isEmpty()) {
        throw new IllegalArgumentException("Fayl bo'sh");
    }
    if (file.getSize() > 5 * 1024 * 1024) {
        throw new IllegalArgumentException("5MB dan katta");
    }

    String contentType = file.getContentType();
    if (!List.of("image/jpeg", "image/png").contains(contentType)) {
        throw new IllegalArgumentException("Faqat JPEG/PNG");
    }

    String fileName = UUID.randomUUID() + "_" + file.getOriginalFilename();
    Path target = Paths.get("uploads", fileName);
    Files.copy(file.getInputStream(), target);

    return Map.of("fileName", fileName, "size", String.valueOf(file.getSize()));
}

// application.yml:
// spring.servlet.multipart.max-file-size: 5MB
// spring.servlet.multipart.max-request-size: 10MB

Custom Response Headers

List API'ga jami soni (X-Total-Count) va cache vaqti (Cache-Control) header'larini qo'shing. 

@GetMapping("/products")
public ResponseEntity<List<Product>> list(
        @RequestParam(defaultValue = "0") int page,
        @RequestParam(defaultValue = "20") int size) {

    Page<Product> result = service.findAll(PageRequest.of(page, size));

    HttpHeaders headers = new HttpHeaders();
    headers.add("X-Total-Count", String.valueOf(result.getTotalElements()));
    headers.add("X-Total-Pages", String.valueOf(result.getTotalPages()));
    headers.setCacheControl(CacheControl.maxAge(5, TimeUnit.MINUTES));

    return ResponseEntity.ok()
        .headers(headers)
        .body(result.getContent());
}

// Response Headers:
// X-Total-Count: 1523
// X-Total-Pages: 77
// Cache-Control: max-age=300

Streaming response — katta CSV

Million qatorli CSV'ni xotiraga to'liq yuklamasdan, oqim ko'rinishida yuboring. 

@GetMapping(value = "/export/users.csv",
            produces = "text/csv")
public ResponseEntity<StreamingResponseBody> exportUsers() {

    StreamingResponseBody stream = output -> {
        try (PrintWriter writer = new PrintWriter(
                new BufferedWriter(new OutputStreamWriter(output)))) {

            // Header
            writer.println("id,name,email,createdAt");

            // Database'dan oqim bilan o'qib chiqish
            try (Stream<User> users = service.streamAll()) {
                users.forEach(u -> {
                    writer.println(String.format("%d,%s,%s,%s",
                        u.getId(), u.getName(),
                        u.getEmail(), u.getCreatedAt()));

                    if (Thread.interrupted()) {
                        return;
                    }
                });
            }
            writer.flush();
        }
    };

    return ResponseEntity.ok()
        .header("Content-Disposition", "attachment; filename=users.csv")
        .body(stream);
}

// Foydasi: 1 mln user uchun ham xotira faqat 1 qatorga ketadi.

CORS — Cross-Origin Resource Sharing

Brauzer xavfsizlik qoidasi (Same-Origin Policy): https://example.com'dan https://api.example.com'ga AJAX so'rov yuborib bo'lmaydi.

CORS — server ruxsat berishi mumkin: "Boshqa origin'lardan ham so'rov qabul qilaman".

Browser avval OPTIONS preflight so'rovi yuboradi va kerakli header'larni tekshiradi.

// 1. Controller darajasida
@RestController
@CrossOrigin(origins = "https://frontend.com")
@RequestMapping("/api/users")
public class UserController { /* ... */ }

// 2. Global configuration (tavsiya)
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins("https://frontend.com", "http://localhost:3000")
            .allowedMethods("GET", "POST", "PUT", "PATCH", "DELETE")
            .allowedHeaders("*")
            .allowCredentials(true)
            .maxAge(3600);  // preflight cache
    }
}
Xavfli: allowedOrigins("*") + allowCredentials(true) birga ishlatish mumkin emas! Aniq domen'lar ko'rsating.
