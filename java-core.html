Java Core

Java dasturlash tilining eng asosiy qismi. Spring Boot, Android yoki Enterprise dasturlashni o'rganishdan oldin Java Core mustahkam bo'lishi shart.

Dars maqsadi: Java dasturining ishlash prinsiplari, o'zgaruvchilar, shart operatorlari, sikllar, massivlar va metodlarni amaliy misollar bilan o'rganib chiqamiz.

Java nima?

Java — obyektga yo'naltirilgan, platformadan mustaqil va kuchli dasturlash tili. 1995-yilda Sun Microsystems tomonidan yaratilgan.

  • Backend dasturlar yozish uchun keng ishlatiladi
  • Spring Boot aynan Java asosida ishlaydi
  • Bank, universitet, ERP, CRM, davlat tizimlarida juda ko'p qo'llaniladi
  • Bir marta yozilgan dastur turli operatsion tizimlarda ishlaydi
  • Andriod ilovalari ham Java/Kotlin bilan yoziladi

JDK, JRE va JVM

JDK

Java Development Kit — Java dasturlarni yozish va ishga tushirish uchun to'liq tool to'plami.

JRE

Java Runtime Environment — tayyor Java dasturlarni ishga tushiruvchi muhit.

JVM

Java Virtual Machine — Java bytecode ni kompyuter tushunadigan ko'rinishga aylantiruvchi virtual mashina.

Oddiy tushuncha: JDK — dasturchi uchun, JRE — foydalanuvchi uchun, JVM — Java kodni ishga tushiruvchi mexanizm.

Birinchi Java dastur

Java dasturi class ichida yoziladi va main metodi orqali ishga tushadi. 

public class Main {
    public static void main(String[] args) {
        System.out.println("Salom, Java!");
    }
}
System.out.println() — ekranga ma'lumot chiqarish uchun ishlatiladi. println — yangi qatordan boshlab chiqaradi, print — yangi qator qo'ymaydi.

Variables — o'zgaruvchilar

O'zgaruvchi — ma'lumot saqlash uchun ajratilgan xotira joyi. Har bir o'zgaruvchining turi va nomi bo'ladi. 

int age = 20;
String name = "Akobir";
double price = 15.5;
boolean active = true;
  • int — butun sonlar
  • String — matn (so'zlar)
  • double — kasr sonlar
  • boolean — true yoki false

Data Types — ma'lumot turlari

Java'da har bir ma'lumot turi xotirada ma'lum hajm egallaydi va aniq diapazonga ega.

  • byte (1 bayt) — butun son: -128 dan +127 gacha
  • short (2 bayt) — butun son: -32 768 dan +32 767 gacha
  • int (4 bayt) — butun son: -2 147 483 648 dan +2 147 483 647 gacha
  • long (8 bayt) — katta butun son: ≈ ±9.2 × 1018
  • float (4 bayt) — kasr son: ≈ ±3.4 × 1038, aniqlik ~7 raqam
  • double (8 bayt) — kasr son: ≈ ±1.7 × 10308, aniqlik ~15 raqam
  • char (2 bayt) — bitta belgi (Unicode): 0 dan 65 535 gacha
  • boolean (1 bit) — faqat true yoki false
long qiymat oxiriga L harfini, float qiymat oxiriga f harfini qo'yish kerak. Masalan: long x = 100000000000L; va float pi = 3.14f;

Arifmetik operatorlar

Arifmetik operatorlar sonlar ustida matematik amal bajaradi. 

int a = 10;
int b = 3;

System.out.println(a + b); // 13  (qo'shish)
System.out.println(a - b); // 7   (ayirish)
System.out.println(a * b); // 30  (ko'paytirish)
System.out.println(a / b); // 3   (bo'lish — butun qism)
System.out.println(a % b); // 1   (qoldiq — modul)
int / int har doim butun son qaytaradi. Kasr olish uchun birini double qiling: (double) a / b → 3.333...

Taqqoslash operatorlari

Taqqoslash operatorlari ikki qiymatni solishtirib, true yoki false qaytaradi. 

int x = 10;
int y = 5;

System.out.println(x == y); // false  (teng)
System.out.println(x != y); // true   (teng emas)
System.out.println(x > y);  // true   (katta)
System.out.println(x < y);  // false  (kichik)
System.out.println(x >= y); // true   (katta yoki teng)
System.out.println(x <= y); // false  (kichik yoki teng)

Mantiqiy operatorlar

Mantiqiy operatorlar bir necha shartni birlashtirib tekshirish uchun ishlatiladi. 

boolean a = true;
boolean b = false;

System.out.println(a && b); // false  (AND — ikkalasi true)
System.out.println(a || b); // true   (OR — kamida bittasi true)
System.out.println(!a);     // false  (NOT — teskari)

int yosh = 20;
boolean kattaYosh = yosh >= 18 && yosh <= 60;
System.out.println(kattaYosh); // true

Kabisa yili (Leap Year)

Berilgan yil kabisa yili ekanligini aniqlang. Yil kabisa yili hisoblanadi, agar:

  • Yil 4 ga bo'linsa, lekin 100 ga bo'linmasa, YOKI
  • Yil 400 ga bo'linsa
Misol
Input: year = 2024 → Output: true
Input: year = 1900 → Output: false
Input: year = 2000 → Output: true 
public static boolean isLeapYear(int year) {
    return (year % 4 == 0 && year % 100 != 0) || year % 400 == 0;
}

System.out.println(isLeapYear(2024)); // true
System.out.println(isLeapYear(1900)); // false
System.out.println(isLeapYear(2000)); // true

Uchburchak yarata olamizmi?

Uchta tomon uzunligi berilgan: a, b, c. Ular bilan uchburchak yasash mumkinmi?

Shart: Har bir tomon qolgan ikkalasining yig'indisidan kichik bo'lishi kerak.

Misol
Input: a=3, b=4, c=5 → Output: true
Input: a=1, b=2, c=10 → Output: false
Input: a=5, b=5, c=5 → Output: true 
public static boolean canFormTriangle(int a, int b, int c) {
    return a + b > c && b + c > a && a + c > b;
}

System.out.println(canFormTriangle(3, 4, 5));  // true
System.out.println(canFormTriangle(1, 2, 10)); // false
System.out.println(canFormTriangle(5, 5, 5));  // true
⚡ Masala 3 Easy

XOR — faqat bittasi rost

Ikkita boolean qiymat berilgan. Aniq bittasi true bo'lsa true qaytaring (XOR amali).

Misol
Input: a=true, b=false → Output: true
Input: a=true, b=true → Output: false
Input: a=false, b=false → Output: false
Eslatma: XOR amalini != orqali ham yozsa bo'ladi, lekin && va || bilan yozish ham mumkin. 
public static boolean exactlyOne(boolean a, boolean b) {
    return (a || b) && !(a && b);
}

System.out.println(exactlyOne(true, false));  // true
System.out.println(exactlyOne(true, true));   // false
System.out.println(exactlyOne(false, false)); // false
⚡ Masala 4 Hard

Uch sonning o'rtanchasi

Uchta turli son berilgan. Faqat taqqoslash va mantiqiy operatorlar yordamida o'rta qiymatni toping (saralash, massivlarsiz).

Misol
Input: a=3, b=7, c=5 → Output: 5
Input: a=10, b=2, c=8 → Output: 8
Input: a=1, b=2, c=3 → Output: 2
G'oya: Son o'rta hisoblanadi, agar u qolgan ikkalasidan biridan katta, ikkinchisidan kichik bo'lsa. 
public static int middle(int a, int b, int c) {
    if ((a >= b && a <= c) || (a <= b && a >= c)) return a;
    if ((b >= a && b <= c) || (b <= a && b >= c)) return b;
    return c;
}

System.out.println(middle(3, 7, 5));  // 5
System.out.println(middle(10, 2, 8)); // 8
System.out.println(middle(1, 2, 3));  // 2

Qiymat berish operatorlari

O'zgaruvchiga qiymat berish va uni qisqa shaklda o'zgartirish. 

int a = 10;

a += 5;  // a = a + 5  → 15
a -= 3;  // a = a - 3  → 12
a *= 2;  // a = a * 2  → 24
a /= 4;  // a = a / 4  → 6
a %= 4;  // a = a % 4  → 2

System.out.println(a); // 2

Inkrement va Dekrement

++ qiymatni 1 ga oshiradi, -- qiymatni 1 ga kamaytiradi. 

int i = 5;

System.out.println(i++); // 5  (avval chiqaradi, keyin oshiradi)
System.out.println(i);   // 6

int j = 5;
System.out.println(++j); // 6  (avval oshiradi, keyin chiqaradi)
System.out.println(j);   // 6

int k = 10;
k--;
System.out.println(k);   // 9
i++ — post-inkrement (avval ishlatadi, keyin oshiradi). ++i — pre-inkrement (avval oshiradi, keyin ishlatadi).

Ternary operator (?:)

Qisqa shart operatori — if/else ning bir qatorli ko'rinishi. 

int yosh = 20;

String natija = (yosh >= 18) ? "Katta yosh" : "Yosh bola";
System.out.println(natija); // Katta yosh

int a = 10, b = 20;
int max = (a > b) ? a : b;
System.out.println(max); // 20
Format: shart ? true_qiymat : false_qiymat

Sonning belgisini aniqlash (Sign of Number)

Berilgan butun son n uchun quyidagi qiymatlardan birini qaytaring:

  • 1 — agar son musbat bo'lsa
  • -1 — agar son manfiy bo'lsa
  • 0 — agar son nol bo'lsa
Misol
Input: n = 25 → Output: 1
Input: n = -7 → Output: -1
Input: n = 0 → Output: 0
Input: n = -1000 → Output: -1
Shart: Faqat ?: (ternary) operatoridan foydalaning. if/else, Math.signum(), switch taqiqlanadi. 
public static int sign(int n) {
    return (n > 0) ? 1 : (n < 0 ? -1 : 0);
}

System.out.println(sign(25));    //  1
System.out.println(sign(-7));    // -1
System.out.println(sign(0));     //  0
System.out.println(sign(-1000)); // -1

FizzBuzz

Klassik LeetCode masala. Berilgan butun son n uchun qaytaring:

  • "FizzBuzz" — agar son 3 ga va 5 ga bo'linsa
  • "Fizz" — agar son faqat 3 ga bo'linsa
  • "Buzz" — agar son faqat 5 ga bo'linsa
  • Sonning o'zi (matn ko'rinishida) — qolgan holatda
Misol
Input: n = 15 → Output: "FizzBuzz"
Input: n = 9 → Output: "Fizz"
Input: n = 10 → Output: "Buzz"
Input: n = 7 → Output: "7"
Maslahat: Avval eng qattiq shartni (ikkalasi ham) tekshiring — tartib muhim! 
public static String fizzBuzz(int n) {
    if (n % 3 == 0 && n % 5 == 0) {
        return "FizzBuzz";
    } else if (n % 3 == 0) {
        return "Fizz";
    } else if (n % 5 == 0) {
        return "Buzz";
    } else {
        return String.valueOf(n);
    }
}

System.out.println(fizzBuzz(15)); // FizzBuzz
System.out.println(fizzBuzz(9));  // Fizz
System.out.println(fizzBuzz(10)); // Buzz
System.out.println(fizzBuzz(7));  // 7

public static String grade(int score) {
    if (score < 0 || score > 100) {
        return "Xato";

        return "A";
    } else if (score >= 80) {
        return "B";
    } else if (score >= 70) {
        return "C";

        return "D";

⚡ Masala 3 Medium

Uchburchak turini aniqlash

Uchta tomon uzunligi berilgan: a, b, c. Uchburchak turini aniqlang:

  • "Teng tomonli" (Equilateral) — barcha tomonlari teng
  • "Teng yonli" (Isosceles) — ikkita tomoni teng
  • "Turli tomonli" (Scalene) — barcha tomonlari turlicha
  • "Uchburchak emas" — uchburchak yarata olmaydi
Misol
Input: 5, 5, 5 → Output: "Teng tomonli"
Input: 5, 5, 8 → Output: "Teng yonli"
Input: 3, 4, 5 → Output: "Turli tomonli"
Input: 1, 2, 10 → Output: "Uchburchak emas"
Eslatma: Uchburchak shart: har bir tomon qolgan ikkalasining yig'indisidan kichik bo'lishi kerak. 
public static String triangleType(int a, int b, int c) {
    if (a + b <= c || b + c <= a || a + c <= b) {
        return "Uchburchak emas";
    }
    if (a == b && b == c) {
        return "Teng tomonli";
    } else if (a == b || b == c || a == c) {
        return "Teng yonli";
    } else {
        return "Turli tomonli";
    }
}

System.out.println(triangleType(5, 5, 5));  // Teng tomonli
System.out.println(triangleType(5, 5, 8));  // Teng yonli
System.out.println(triangleType(3, 4, 5));  // Turli tomonli
System.out.println(triangleType(1, 2, 10)); // Uchburchak emas
⚡ Masala 4 Hard

Progressiv daromad solig'i

Yillik daromadga qarab soliq miqdorini hisoblang (bosqichli soliq tizimi):

  • 0 — 10 000 000 so'm: 0% (soliq yo'q)
  • 10M — 50M so'm: 10M dan ortig'ining 12%i
  • 50M — 200M so'm: avvalgi + 50M dan ortig'ining 20%i
  • 200M dan yuqori: avvalgi + 200M dan ortig'ining 30%i
Misol
Input: 8 000 000 → Output: 0
Input: 25 000 000 → Output: 1 800 000 (15M × 12%)
Input: 100 000 000 → Output: 14 800 000 (4.8M + 50M × 20%)
Input: 300 000 000 → Output: 64 800 000
Maslahat: Har bir bosqichni alohida hisoblang — pastdan yuqoriga qarab. 
public static long calculateTax(long income) {
    if (income <= 10_000_000) {
        return 0;
    } else if (income <= 50_000_000) {
        return (long) ((income - 10_000_000) * 0.12);
    } else if (income <= 200_000_000) {
        long base = (long) (40_000_000 * 0.12); // 4 800 000
        return base + (long) ((income - 50_000_000) * 0.20);
    } else {
        long base1 = (long) (40_000_000 * 0.12);   // 4 800 000
        long base2 = (long) (150_000_000 * 0.20);  // 30 000 000
        return base1 + base2 + (long) ((income - 200_000_000) * 0.30);
    }
}

System.out.println(calculateTax(8_000_000));   // 0
System.out.println(calculateTax(25_000_000));  // 1 800 000
System.out.println(calculateTax(100_000_000)); // 14 800 000
System.out.println(calculateTax(300_000_000)); // 64 800 000

Faktorial (n!)

Berilgan musbat butun son n ning faktorialini hisoblang.

Faktorial — 1 dan n gacha bo'lgan barcha sonlarning ko'paytmasi:

n! = 1 × 2 × 3 × ... × n

Misol
Input: n = 5 → Output: 120 (1×2×3×4×5)
Input: n = 1 → Output: 1
Input: n = 0 → Output: 1 (matematik kelishuv)
Input: n = 10 → Output: 3 628 800
Eslatma: Natija juda tez katta bo'ladi, shuning uchun long turidan foydalaning. 20! dan keyin long ham yetmaydi. 
public static long factorial(int n) {
    long result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}

System.out.println(factorial(5));  // 120
System.out.println(factorial(1));  // 1
System.out.println(factorial(0));  // 1
System.out.println(factorial(10)); // 3628800

Sonning raqamlari yig'indisi

Berilgan musbat butun son n ning raqamlari yig'indisini toping.

Misol
Input: n = 1234 → Output: 10 (1+2+3+4)
Input: n = 9 → Output: 9
Input: n = 99999 → Output: 45
Input: n = 1000000 → Output: 1
G'oya: n % 10 oxirgi raqamni beradi, n / 10 oxirgi raqamni olib tashlaydi. n > 0 bo'lguncha takrorlaymiz. 
public static int digitSum(int n) {
    int sum = 0;
    while (n > 0) {
        sum += n % 10;  // oxirgi raqamni qo'shamiz
        n /= 10;        // oxirgi raqamni olib tashlaymiz
    }
    return sum;
}

System.out.println(digitSum(1234));    // 10
System.out.println(digitSum(9));       // 9
System.out.println(digitSum(99999));   // 45
System.out.println(digitSum(1000000)); // 1
⚡ Masala 3 Medium

Tub son tekshirish (Prime Number)

Berilgan son n tub son ekanligini aniqlang.

Tub son — 1 dan katta, faqat 1 ga va o'ziga bo'linadigan son.

Misol
Input: n = 7 → Output: true (faqat 1 va 7 ga bo'linadi)
Input: n = 12 → Output: false (1, 2, 3, 4, 6, 12)
Input: n = 2 → Output: true
Input: n = 1 → Output: false
Optimizatsiya: 2 dan √n gacha tekshirish yetarli — agar n ning bo'luvchisi bor bo'lsa, u albatta √n dan kichik bo'ladi. 
public static boolean isPrime(int n) {
    if (n < 2) return false;
    if (n == 2) return true;
    if (n % 2 == 0) return false;

    for (int i = 3; i * i <= n; i += 2) {
        if (n % i == 0) {
            return false;
        }
    }
    return true;
}

System.out.println(isPrime(7));  // true
System.out.println(isPrime(12)); // false
System.out.println(isPrime(2));  // true
System.out.println(isPrime(1));  // false

📝 Amaliy topshiriqlar Mustaqil

10 ta murakkab vazifa

Quyidagi 10 ta vazifani o'zingiz mustaqil yechib ko'ring. Har biri bir nechta tushunchani birlashtiradi:

  • 1. Ikkinchi eng katta son. Massiv ichidan saralamasdan ikkinchi maksimal sonni toping.
    [3, 7, 2, 9, 5] → 7   |   [10, 10, 8] → 8
  • 2. Massivni teskari aylantirish (in-place). Yangi massiv yaratmasdan, joyida o'zgartiring.
    [1, 2, 3, 4, 5] → [5, 4, 3, 2, 1]
  • 3. Armstrong soni. Son raqamlari kublari yig'indisi o'ziga teng bo'lsa, Armstrong sonidir.
    153 → true (1³+5³+3³=153)   |   123 → false
  • 4. GCD (Evklid algoritmi). Ikki sonning eng katta umumiy bo'luvchisini toping.
    gcd(48, 18) → 6   |   gcd(100, 75) → 25
  • 5. Raqamli ildiz (Digital Root). Sonning raqamlarini bitta raqam qolguncha qo'shing.
    9875 → 9+8+7+5=29 → 2+9=11 → 1+1=2 → natija: 2
  • 6. Fibonacci ketma-ketligi. N-elementni rekursiyasiz (iterativ) toping.
    fib(10) → 55   |   fib(20) → 6765
  • 7. Palindrom tekshirish. String teskari o'qiganda ham bir xil bo'lsa, palindrom.
    "level" → true   |   "java" → false
  • 8. Bubble Sort. Massivni qo'lda saralash algoritmini yozing (kutubxonasiz).
    [5, 2, 8, 1, 9] → [1, 2, 5, 8, 9]
  • 9. Two Sum (LeetCode klassik). Massivda yig'indisi target ga teng ikki son indekslarini toping.
    nums=[2, 7, 11, 15], target=9 → [0, 1]
  • 10. Pascal uchburchagi. N-satrini chiqaring.
    row(4) → [1, 4, 6, 4, 1]   |   row(0) → [1]
Har bir vazifa uchun: alohida metod yozing, kamida 3 ta test holatini sinab ko'ring va kod toza, o'qiladigan bo'lsin.
